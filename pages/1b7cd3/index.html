Constructors and initialization of derived classes

# Constructors and initialization of derived classes

在这节课中，我们将进一步了解构造函数在派生类初始化中的作用。为此，我们将继续使用在前一课中开发的简单基类和派生类:

class Base
{
public:
    int m_id;
 
    Base(int id=0)
        : m_id{ id }
    {
    }
 
    int getId() const { return m_id; }
};
 
class Derived: public Base
{
public:
    double m_cost;
 
    Derived(double cost=0.0)
        : m_cost{ cost }
    {
    }
 
    double getCost() const { return m_cost; }
};

  4. The body of the constructor executes

int main()

下面是派生类被实例化时实际发生的情况:

  1. Memory for derived is set aside (enough for both the Base and Derived portions)
  2. The appropriate Derived constructor is called
  3. The Base object is constructed first using the appropriate Base constructor. If no base constructor is specified, the default constructor will be used.
  4. The initialization list initializes variables
  5. The body of the constructor executes
  6. Control is returned to the caller

这种情况和非继承情况之间的唯一真正区别是，在派生构造函数可以在执行任何实质操作之前，首先调用基构造函数。基类构造函数设置对象的基类部分，将控制权返回给派生类构造函数，并允许派生类构造函数完成其工作。

# Initializing base class members

目前所写的派生类的一个缺点是，当我们创建派生类对象时，没有办法初始化m_id。当我们创建一个派生对象时， 如果我们想设置m_cost(从对象的派生部分)和m_id(从对象的基部分)，我们应该如何做呢？

新程序员经常试图解决这个问题如下:

class Derived: public Base
{
public:
    double m_cost;
 
    Derived(double cost=0.0, int id=0)
        // does not work
        : m_cost{ cost }, m_id{ id }
    {
    }
 
    double getCost() const { return m_cost; }
}

这是一个很好的尝试，而且几乎是一个正确的想法。我们肯定需要向构造函数添加另一个形参，否则c++将无法知道我们想将m_id初始化为什么值。

但是，c++阻止类在构造函数的初始化列表中初始化继承的成员变量。换句话说，成员变量的值只能在与该变量属于同一类的构造函数的初始化列表中设置。

C++为什么要这样做呢?答案与const和引用变量有关。考虑一下如果m_id是const会发生什么。因为const变量必须在创建时用值初始化，所以基类构造函数必须在创建变量时设置其值。但是，当基类构造函数完成时，将执行派生类构造函数初始化列表。然后每个派生类将有机会初始化该变量，可能会改变其值!通过将变量的初始化限制在类的构造函数中，c++确保所有变量只初始化一次。

最终的结果是，上面的例子不能工作，因为m_id是从Base继承的，并且只有非继承的变量可以在初始化列表中初始化。

但是，仍然可以使用赋值在构造函数体中更改继承变量的值。因此，新程序员通常也会尝试这样做:

class Derived: public Base
{
public:
    double m_cost;
 
    Derived(double cost=0.0, int id=0)
        : m_cost{ cost }
    {
        m_id = id;
    }
 
    double getCost() const { return m_cost; }
};

虽然这在这种情况下实际上是有效的，但如果m_id是一个const或引用，它就不起作用了(因为const值和引用必须在构造函数的初始化列表中初始化)。这也是低效的，因为m_id被赋值两次:一次在基类构造函数的初始化列表中，然后在派生类构造函数的主体中。最后，如果基类在构造过程中需要访问这个值怎么办?它没有办法访问它，因为直到派生构造函数被执行(这几乎是最后一次执行)，它才被设置。

当创建一个派生类对象时，我们如何正确初始化m_id ?

在迄今为止的所有示例中，当实例化Derived类对象时，已经使用默认的Base构造函数创建了Base类部分。为什么它总是使用默认的Base构造函数?因为我们从没告诉过它不这样做!

幸运的是，c++提供了显式选择调用哪个基类构造函数的能力!要做到这一点，只需在派生类的初始化列表中添加对基类Constructor的调用:

class Base
{
public:
    int m_id;
 
    Base(int id=0)
        : m_id{ id }
    {
    }
 
    int getId() const { return m_id; }
};
class Derived: public Base
{
public:
    double m_cost;
 
    Derived(double cost=0.0, int id=0)
        : Base{ id }, // Call Base(int) constructor with value id!
            m_cost{ cost }
    {
    }
 
    double getCost() const { return m_cost; }
};

int main()
{
    Derived derived{ 1.3, 5 }; // use Derived(double, int) constructor
    std::cout << "Id: " << derived.getId() << '\n';
    std::cout << "Cost: " << derived.getCost() << '\n';
 
    return 0;
}

基类构造函数base (int)将用于初始化m_id为5，派生类构造函数将用于初始化m_cost为1.3!

下面是派生被实例化时实际发生的情况:

  1. Memory for derived is allocated.
  2. The Derived(double, int) constructor is called, where cost = 1.3, and id = 5
  3. The compiler looks to see if we’ve asked for a particular Base class constructor. We have! So it callsBase(int) with id = 5.
  4. The base class constructor initialization list sets m_id to 5
  5. The base class constructor body executes, which does nothing
  6. The base class constructor returns
  7. The derived class constructor initialization list sets m_cost to 1.3
  8. The derived class constructor body executes, which does nothing
  9. The derived class constructor returns

这看起来有点复杂，但实际上非常简单。所发生的一切只是派生构造函数调用特定的Base构造函数来初始化对象的Base部分。因为m_id存在于对象的Base部分，所以Base构造函数是唯一可以初始化该值的构造函数。

请注意，在派生构造函数初始化列表的哪个位置调用Base构造函数并不重要——它总是首先执行。

# Now we can make our members private

既然您知道了如何初始化基类成员，就没有必要将成员变量保持为public。我们再次将成员变量设置为私有，它们应该是私有的。

任何人都可以访问公共成员。私有成员只能由同一类的成员函数访问。注意，这意味着派生类不能直接访问基类的私有成员!派生类将需要使用访问函数来访问基类的私有成员。

#include <iostream>
 
class Base
{
private: // our member is now private
    int m_id;
 
public:
    Base(int id=0)
        : m_id{ id }
    {
    }
 
    int getId() const { return m_id; }
};
 
class Derived: public Base
{
private: // our member is now private
    double m_cost;
 
public:
    Derived(double cost=0.0, int id=0)
        : Base{ id }, // Call Base(int) constructor with value id!
            m_cost{ cost }
    {
    }
 
    double getCost() const { return m_cost; }
};
 
int main()
{
    Derived derived{ 1.3, 5 }; // use Derived(double, int) constructor
    std::cout << "Id: " << derived.getId() << '\n';
    std::cout << "Cost: " << derived.getCost() << '\n';
 
    return 0;
}

正如我们之前所写的，BaseballPlayer只初始化它自己的成员，而不指定要使用的Person构造函数。这意味着我们创建的每个BaseballPlayer都将使用默认的Person构造函数，该构造函数将初始化名称为空白，年龄为0。因为在创建BaseballPlayer时给它一个名称和年龄是有意义的，所以我们应该修改这个构造函数来添加这些参数。

下面是我们使用私有成员的更新类，BaseballPlayer类调用适当的Person构造函数来初始化继承的Person成员变量:

#include <iostream>
#include <string>
 
class Person
{
private:
    std::string m_name;
    int m_age;
 
public:
    Person(const std::string& name = "", int age = 0)
        : m_name{ name }, m_age{ age }
    {
    }
 
    const std::string& getName() const { return m_name; }
    int getAge() const { return m_age; }
 
};
// BaseballPlayer publicly inheriting Person
class BaseballPlayer : public Person
{
private:
    double m_battingAverage;
    int m_homeRuns;
 
public:
    BaseballPlayer(const std::string& name = "", int age = 0,
        double battingAverage = 0.0, int homeRuns = 0)
        : Person{ name, age }, // call Person(const std::string&, int) to initialize these fields
            m_battingAverage{ battingAverage }, m_homeRuns{ homeRuns }
    {
    }
 
    double getBattingAverage() const { return m_battingAverage; }
    int getHomeRuns() const { return m_homeRuns; }
};

# Inheritance chains

#include <iostream>
 
class A
{
public:
    A(int a)
    {
        std::cout << "A: " << a << '\n';
    }
};
 
class B: public A
{
public:
    B(int a, double b)
    : A{ a }
    {
        std::cout << "B: " << b << '\n';
    }
};
 
class C: public B
{
public:
    C(int a , double b , char c)
    : B{ a, b }
    {
        std::cout << "C: " << c << '\n';
    }
};
 
int main()
{
    C c{ 5, 4.3, 'R' };
 
    return 0;
}

在这个例子中，类C是从类B派生出来的，而类B又是从类a派生出来的。那么，当我们实例化类C的一个对象时会发生什么?

首先，main()调用C(int, double, char)。C构造函数调用B(int, double)。B构造函数调用A(int)。因为A不继承任何人，这是我们要构造的第一个类。构造A，打印值5，并将控制权返回给B。构造B，打印值4.3，并将控制权返回给C。构造C，打印值' R '，并将控制权返回给main()。我们完成了!

值得一提的是，构造函数只能从它们的直接父类/基类调用构造函数。因此，C构造函数不能直接调用或传递参数给A构造函数。C构造函数只能调用B构造函数(B构造函数负责调用A构造函数)。

# Destructors

当销毁派生类时，将按照构造的相反顺序调用每个析构函数。在上面的例子中，当c被撤销时，首先调用c析构函数，然后调用B析构函数，然后调用A析构函数。

# Summary

在构造派生类时，派生类构造函数负责确定调用了哪个基类构造函数。如果没有指定基类构造函数，将使用默认基类构造函数。在这种情况下，如果找不到默认的基类构造函数(或默认创建)，编译器将显示一个错误。然后按照从最基类到最派生类的顺序构造这些类。
