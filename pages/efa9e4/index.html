Move constructors and move assignment

# Move constructors and move assignment

在这节课中，我们将深入了解c++ 11如何通过move构造函数和move赋值来解决这些问题。

# Copy constructors and copy assignment

首先，让我们花点时间来回顾一下复制语义。

复制构造函数通过复制同一个类的一个对象来初始化一个类。复制赋值用于将一个类对象复制到另一个现有的类对象。默认情况下，c++将提供复制构造函数和复制赋值操作符(如果没有显式提供)。这些编译器提供的函数进行浅拷贝，这可能会给分配动态内存的类带来问题。因此，处理动态内存的类应该重写这些函数来进行深度复制。

回到本章第一课的Auto_ptr智能指针类示例，让我们看一个实现了复制构造函数和复制赋值操作符的版本，以及一个演示它们的示例程序:

template<class T>
class Auto_ptr3
{
	T* m_ptr;
public:
	Auto_ptr3(T* ptr = nullptr)
		:m_ptr(ptr)
	{
	}
 
	~Auto_ptr3()
	{
		delete m_ptr;
	}
 
	// Copy constructor
	// Do deep copy of a.m_ptr to m_ptr
	Auto_ptr3(const Auto_ptr3& a)
	{
		m_ptr = new T;
		*m_ptr = *a.m_ptr;
	}
 
	// Copy assignment
	// Do deep copy of a.m_ptr to m_ptr
	Auto_ptr3& operator=(const Auto_ptr3& a)
	{
		// Self-assignment detection
		if (&a == this)
			return *this;
 
		// Release any resource we're holding
		delete m_ptr;
 
		// Copy the resource
		m_ptr = new T;
		*m_ptr = *a.m_ptr;
 
		return *this;
	}
 
	T& operator*() const { return *m_ptr; }
	T* operator->() const { return m_ptr; }
	bool isNull() const { return m_ptr == nullptr; }
};
 
class Resource
{
public:
	Resource() { std::cout << "Resource acquired\n"; }
	~Resource() { std::cout << "Resource destroyed\n"; }
};
 
Auto_ptr3<Resource> generateResource()
{
	Auto_ptr3<Resource> res(new Resource);
	return res; // this return value will invoke the copy constructor
}
 
int main()
{
	Auto_ptr3<Resource> mainres;
	mainres = generateResource(); // this assignment will invoke the copy assignment
 
	return 0;
}
/*
Resource acquired 第一次创建资源
Resource acquired 通过值返回时，创建了临时对象
Resource destroyed 函数里的局部变量res超出范围
Resource acquired 赋值构造函数中，首先创建一个新变量
Resource destroyed 然后删除原有的的
Resource destroyed 最后是main函数里面的
*/

在这个程序中，我们使用一个名为generateResource()的函数来创建一个封装资源的智能指针，然后将其传递回函数main()。然后函数main()将其赋值给一个现有的Auto_ptr3对象。

(注意:如果你的编译器省略了函数generateResource()的返回值，你可能只会得到4个输出)

这样一个简单的程序需要大量的资源创建和destory!这是怎么回事?

让我们仔细看看。在这个程序中有6个关键步骤(每个打印信息一个):

  1. 在generateResource()中，使用动态分配的Resource创建并初始化局部变量res，这将导致第一个“Resource acquired”。
  2. Res按值返回给main()。这里我们按值返回，因为res是一个局部变量——它不能按地址或引用返回，因为当generateResource()结束时res将被销毁。所以res是复制构造到一个临时对象。因为我们的复制构造函数进行了深层复制，所以在这里分配了一个新的Resource，这导致了第二个“Resource acquired”。
  3. Res超出范围，破坏最初创建的资源，导致第一个“Resource destroyed”。
  4. 临时对象是通过复制赋值的方式分配给mainres的。因为我们的拷贝赋值也会进行深层拷贝，所以会分配一个新的Resource，从而导致另一个“Resource acquired”。
  5. 赋值表达式结束，临时对象超出表达式范围并被销毁，导致“Resource acquired”。
  6. 在main()的末尾，mainres超出作用域，最后显示“Resource destroyed”。

因此，简而言之，因为我们调用复制构造函数一次，将construct res复制到一个临时对象，并调用赋值函数一次，将临时对象复制到main对象中，所以我们最终总共分配和销毁了3个单独的对象。

效率低下，但至少不会崩溃!

然而，通过移动语义，我们可以做得更好。

# Move constructors and move assignment

c++ 11为move语义定义了两个新函数:一个move构造函数和一个move赋值操作符。而复制构造函数和复制的目标任务是复制一个对象到另一个地方,移动构造函数和移动的目标任务是对资源的所有权从一个对象转移到另一个(通常是比复制便宜得多)。

定义一个move构造函数和move赋值操作类似于它们的副本。然而，这些函数的复制类型使用const l值引用形参，而移动类型使用非const r值引用形参。

下面是与上面相同的Auto_ptr3类，添加了一个move构造函数和move赋值操作符。为了进行比较，我们保留了深度复制的复制构造函数和复制赋值操作符。

#include <iostream>
 
template<class T>
class Auto_ptr4
{
	T* m_ptr;
public:
	Auto_ptr4(T* ptr = nullptr)
		:m_ptr(ptr)
	{
	}
 
	~Auto_ptr4()
	{
		delete m_ptr;
	}
 
	// Copy constructor
	// Do deep copy of a.m_ptr to m_ptr
	Auto_ptr4(const Auto_ptr4& a)
	{
		m_ptr = new T;
		*m_ptr = *a.m_ptr;
	}
 
	// Move constructor
	// Transfer ownership of a.m_ptr to m_ptr
	Auto_ptr4(Auto_ptr4&& a) noexcept
		: m_ptr(a.m_ptr)
	{
		a.m_ptr = nullptr; // we'll talk more about this line below
	}
 
	// Copy assignment
	// Do deep copy of a.m_ptr to m_ptr
	Auto_ptr4& operator=(const Auto_ptr4& a)
	{
		// Self-assignment detection
		if (&a == this)
			return *this;
 
		// Release any resource we're holding
		delete m_ptr;
 
		// Copy the resource
		m_ptr = new T;
		*m_ptr = *a.m_ptr;
 
		return *this;
	}
 
	// Move assignment
	// Transfer ownership of a.m_ptr to m_ptr
	Auto_ptr4& operator=(Auto_ptr4&& a) noexcept
	{
		// Self-assignment detection
		if (&a == this)
			return *this;
 
		// Release any resource we're holding
		delete m_ptr;
 
		// Transfer ownership of a.m_ptr to m_ptr
		m_ptr = a.m_ptr;
		a.m_ptr = nullptr; // we'll talk more about this line below
 
		return *this;
	}
 
	T& operator*() const { return *m_ptr; }
	T* operator->() const { return m_ptr; }
	bool isNull() const { return m_ptr == nullptr; }
};
 
class Resource
{
public:
	Resource() { std::cout << "Resource acquired\n"; }
	~Resource() { std::cout << "Resource destroyed\n"; }
};
 
Auto_ptr4<Resource> generateResource()
{
	Auto_ptr4<Resource> res(new Resource);
	return res; // this return value will invoke the move constructor
}
 
int main()
{
	Auto_ptr4<Resource> mainres;
	mainres = generateResource(); // this assignment will invoke the move assignment
 
	return 0;
}
