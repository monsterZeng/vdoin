Distance

Reference:

  • Different types of Distances used in Machine Learning (opens new window)

在数学、统计学和机器学习领域，“距离度量”已经有了各种各样的定义。但是，这些术语、概念和它们的用法非常的广泛。距离度量在机器学习中非常重要，以做出一个好的基于数据的决策。选择一个好的距离度量对于识别内容之间的相似性很重要。我的这个博客讨论了不同类型的距离及其在机器学习中的应用。

# Minkowski Distance

为了定义闵可夫斯基距离，我们首先学习一下一些数学定义

  • 向量空间：它是被称为向量的对象的集合，这些对象可以被数字相加和相乘并且不超出这个空间。
  • 范数：范数是一个函数，它负责分配一个严格为正的长度给向量空间中的每个向量。通常表示为：||x||
  • 赋范线性空间：它是一个定义范数在实数或复数上的向量空间。

闵可夫斯基距离定义为赋范向量空间中两点之间的相似性度量。它由下面的公式表示，

D(x,y)=(∑i=1n∣xi−yi∣)1/pD(x,y)=\left(\sum_{i=1}^{n}|x_i-y_i|\right)^{1/p} D(x,y)=(​i=1​∑​n​​∣x​i​​−y​i​​∣)​1/p​​

它还代表了一个广义度量，包括欧几里得距离和曼哈顿距离。我们可以用三种不同的方法来计算p的值和距离，

  • p=1，曼哈顿距离
  • p=2，欧式距离
  • p=∞\infty∞​，切比雪夫距离

# Where it is used?

Minkowski distance is frequently used when the variables of interest are measured on ratio scales with an absolute zero value.❓

# Euclidean Distance

当我们谈论距离时，我们通常认为它是一条或长或短的直线。如果我们考虑从一个城市飞到另一个城市，我们会考虑乘飞机要走多少公里。这些距离的例子都是欧几里得距离的例子。

基本上，它测量的是连接两点的线段的长度。例如勾股定理

勾股定理给出了两点之间的距离。将p=2代入闵可夫斯基距离公式，即可得到欧几里得距离方程。它也被称为L2范数。

D(x,y)=(∑i=1n∣xi−yi∣)1/2D(x,y)=\left(\sum_{i=1}^{n}|x_i-y_i|\right)^{1/2} D(x,y)=(​i=1​∑​n​​∣x​i​​−y​i​​∣)​1/2​​

在某些情况下欧几里得距离不能给出合适的度量。在这些情况下，我们需要利用下面提到的距离函数。

# Manhattan Distance

假设我们想计算城市中两个街区之间的距离。这一次，我们将在一个网格状的路径中计算距离，它代表了城市中的各个街区。

假设，我们想从一个城市的A区到B区。从A街区到B街区的距离叫做曼哈顿距离。将p=1代入闵可夫斯基距离的公式，即可得到曼哈顿距离的方程。它也叫做L1范数。

D(x,y)=(∑i=1n∣xi−yi∣)1/1D(x,y)=\left(\sum_{i=1}^{n}|x_i-y_i|\right)^{1/1} D(x,y)=(​i=1​∑​n​​∣x​i​​−y​i​​∣)​1/1​​

曼哈顿距离也被称为出租车几何，城市街区距离等。

Where Euclidean and Manhattan distance is used?

曼哈顿距离和欧几里得距离都被用于回归和分类问题。但欧几里得距离在高维数据中并不适用。这是由于所谓的“维度诅咒”而发生的。维度诅咒是指在高维空间(通常有数百或数千个维度)中分析和组织数据时出现的各种现象，而在低维环境(如日常经验的三维物理空间)中不会出现这些现象。在高维数据中，曼哈顿距离是首选。 此外，如果您正在计算误差，曼哈顿距离在您想要强调其线性性质的异常值时非常有用。

# Hamming Distance

汉明距离是比较两个二进制数据串的度量。当比较两个相同长度的二进制串时，汉明距离是两个位不相同的位的位数。两个字符串a和b之间的汉明距离记为d(a,b)。

// leetcode: 461.汉明距离
// 朴素解法
class Solution {
public:
    int hammingDistance(int x, int y) {
        int cnt = 0;
        while(x || y){
            if((x & 0x01) != (y & 0x01)){
                cnt++;
            }
            x>>=1;
            y>>=1;
        }
        return cnt;
    }
};
class Solution {
public:
    int hammingDistance(int x, int y) {
        int cnt = 0, z = x ^ y;
        while(z){
            if(z&0x01){
                cnt++;
            }
            z>>=1;
        }
        return cnt;
    }
};

# Cosine Distance and Cosine Similarity

为了确定两个文档或任何类型的实体有多相似，我们使用余弦相似度。为了计算它，我们需要测量两个向量夹角的余弦值。余弦距离和余弦相似性是成反比的。如果余弦距离增大，余弦相似度减小，反之亦然。因此，两个相距较近的点比相距较远的点具有相似的性质。

Cosine similarity =cosθ

Cosine distance =1-cosθ.

在上图中，两条线A和B的夹角是45°。因此余弦相似度是cos45°，大约是0.53，这意味着这些点有53%的相似度。余弦距离是(1-0.53)等于0.47。余弦相似度的值必须在-1到+1之间。

Where cosine distance and cosine similarity is used?

余弦距离和余弦相似度用于推荐系统。

让我们来看看电影推荐系统。以“动作片”为x轴，以“喜剧”为y轴，让我们以两部电影为例，《复仇者联盟》属于“动作片”，《卑鄙的我》属于“喜剧”。所以我们可以根据电影的名字分别画出点(0,1)和(1,0)。点(0,1)意味着这部电影没有太多的动作，但更多的是喜剧。因此，两点相似度为0，即cos90°。因此，电影推荐系统不会推荐喜欢“喜剧”类型电影的人，而是推荐喜欢“动作片”类型电影的人。

同样的，如果我们把《卑鄙的我》和《玩具总动员》这样的电影在y轴上的坐标是(0,0.9)和(0,1)，它们之间的角度是0°，也就是说余弦相似度是1(cos 0°)。因此，如果用户已经看过《玩具总动员》，系统会推荐他去看《卑鄙的我》。

# 证明余弦距离并不是严格定义的距离

严格定义的距离需要满足三个特性

  • 正定性，距离必定大于零
  • 对称性，dist(A,B)=dist(B,A)
  • 三角不等式，dist(A, B) + dist(B, C)>= dist(A, C)

证明正定性

cosdistance=1−cosθ>=0\text{cos distance} = 1-\cos\theta >= 0 cosdistance=1−cosθ>=0

由式子可得，余弦距离的范围在[0,2][0,2][0,2]之间，所以满足正定性

证明对称性。
