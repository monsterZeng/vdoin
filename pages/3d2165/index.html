最大化最小值

Farmer John has built a new long barn, with N (2 <= N <= 100,000) stalls. The stalls are located along a straight line at positions x~1~,…,x~N~ (0 <= x~i~ <= 1,000,000,000).

His C (2 <= C <= N) cows don’t like this barn layout and become aggressive towards each other once put into a stall. To prevent the cows from hurting each other, FJ want to assign the cows to the stalls, such that the minimum distance between any two of them is as large as possible. What is the largest minimum distance?

​ 农夫John修建了一个很长的畜棚，有N个畜栏。这些畜栏都被放置到一个直线上，位置为x~1~,...,x~N~(0<=x~i~<=1000,000,000)。他有C头奶牛，但是并不喜欢它这种畜栏的设计，一旦把这些牛塞到同一个畜栏里就会发狂。为了阻止这些奶牛伤害到彼此，农夫John想要去这些奶牛分配到任意两头牛之间的最小距离尽可能大。这个最小距离是什么？

1552. 两球之间的磁力 (opens new window)

在代号为 C-137 的地球上，Rick 发现如果他将两个球放在他新发明的篮子里，它们之间会形成特殊形式的磁力。Rick 有 n 个空的篮子，第 i 个篮子的位置在 position[i] ，Morty 想把 m 个球放到这些篮子里，使得任意两球间 最小磁力 最大。

已知两个球如果分别位于 x 和 y ，那么它们之间的磁力为 |x - y| 。

给你一个整数数组 position 和一个整数 m ，请你返回最大化的最小磁力。
输入：position = [1,2,3,4,7], m = 3
输出：3
解释：将 3 个球分别放入位于 1，4 和 7 的三个篮子，两球间的磁力分别为 [3, 3, 6]。最小磁力为 3 。我们没办法让最小磁力大于 3 。

/*
	两个球之间的磁力为|x - y|，我们希望使得任意两个球之间的最小磁力最大，也就是最大化最小磁力。
	磁力值最小为1，最大为两个position数组中最大值与最小值之间的差值。
	由于position中的值为坐标，所以需要排序。
	迭代时，在区间选取一个距离，看在此距离下是否能够容纳m个球。
	如果可以，那么增大距离；如果不行，就减少距离
*/
class Solution {
public:
    int maxDistance(vector<int>& position, int m) {
        sort(position.begin(), position.end());
        int n = position.size();
        int min = 1, max = position[n - 1] - position[0];
        int res = INT_MIN;
        while (min <= max) {
            int c = (max - min) / 2 + min;
            int tmp = check(position, c, m);
            if (tmp == -1) { // 说明这个距离不能装下m个球，需要小一点距离
                max = c - 1;
            }
            else { // 这个距离可以大一点
                res = (res < tmp) ? tmp : res; // 不直接使用c的原因就是因为坐标可能不连续，磁力可能不刚好
                min = c + 1;
            }
        }
        return res;

    }
    // 要检查的是，在至少相隔距离为dist，返回两个球之间的最小距离，如果放不下m个球，就返回-1
    int check(vector<int>& position, int dist, int m) {
        int n = position.size();
        int left = position[0];
        int res = INT_MAX;
        m--;
        for (int i = 1; i < n && m != 0; i++) {
            if (abs(position[i] - left) >= dist) { //说明这两个盒子之间能放球
                res = min(res, abs(position[i] - left));
                left = position[i];
                m--;
            }
        }
        return (m == 0) ? res : -1;
    }

};

# 1898. 可移除字符的最大数目 (opens new window)

给你两个字符串 s 和 p ，其中 p 是 s 的一个 子序列 。同时，给你一个元素 互不相同 且下标 从 0 开始 计数的整数数组 removable ，该数组是 s 中下标的一个子集（s 的下标也 从 0 开始 计数）。

请你找出一个整数 k（0 <= k <= removable.length），选出 removable 中的 前 k 个下标，然后从 s 中移除这些下标对应的 k 个字符。整数 k 需满足：在执行完上述步骤后， p 仍然是 s 的一个 子序列 。更正式的解释是，对于每个 0 <= i < k ，先标记出位于 s[removable[i]] 的字符，接着移除所有标记过的字符，然后检查 p 是否仍然是 s 的一个子序列。

返回你可以找出的 最大 k ，满足在移除字符后 p 仍然是 s 的一个子序列。

字符串的一个 子序列 是一个由原字符串生成的新字符串，生成过程中可能会移除原字符串中的一些字符（也可能不移除）但不改变剩余字符之间的相对顺序。

输入：s = "abcacb", p = "ab", removable = [3,1,0]
输出：2
解释：在移除下标 3 和 1 对应的字符后，"abcacb" 变成 "accb" 。
"ab" 是 "accb" 的一个子序列。
如果移除下标 3、1 和 0 对应的字符后，"abcacb" 变成 "ccb" ，那么 "ab" 就不再是 s 的一个子序列。
因此，最大的 k 是 2 。

/*
	对于可移除的字符个数k，最小为0，最大为removable.size()。
	假设能移除K个字符，那么查看移除K个字符后，字符串p是否还是s的子序列。
	如果是，那么增大搜索范围；如果不是，那么减小搜索空间。
*/
class Solution {
public:
    int maximumRemovals(string s, string p, vector<int>& removable) {
        int left = 0, right = removable.size();
        int res = INT_MIN;
        while(left <= right){
            int c = (right - left) / 2 + left;
            if(check(s, p, removable, c)){
                left = c + 1;
                res = max(res, c);
            }else{
                right = c - 1;
            }
        }
        return res;
    }
    /*
        查看移除在removable前k个索引后的字符串s，p还是s的子序列
    */
    bool check(string s, string p, vector<int>& removable, int k){
        // s移除字符
        for(int i = 0; i < k; i ++){
            s[removable[i]] = '#';
        }
        // 检验p是否为s的子序列
        int j = 0;
        for(int i = 0; i < s.size() && j != p.size(); i ++){
            if(s[i] == p[j]){
                j++;
            }
            if(j == p.size()) return true;
        }
        return false;
    }
};
