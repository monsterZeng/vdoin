# Function overloading

函数重载是C ++的一项特征，它允许我们创建具有相同名称的多个函数，只要它们具有不同的参数即可。

考虑以下函数：

int add(int x, int y)
{
    return x + y;
}
1
2
3
4

这个简单的函数将两个整数相加。但是，如果我们还需要添加两个浮点数怎么办？此函数根本不合适，因为任何浮点参数都将转换为整数，从而导致浮点参数丢失其小数值。

解决此问题的一种方法是定义名称稍有不同的多个函数：

int addInteger(int x, int y)
{
    return x + y;
}
double addDouble(double x, double y)
{
    return x + y;
}
1
2
3
4

8

但是，为了获得最佳效果，这要求您保持一致的命名标准，记住该函数所有不同样式的名称，然后调用正确的名称。 函数重载提供了更好的解决方案。使用函数重载，我们可以简单地声明另一个采用double的add()函数：

double add(double x, double y)
{
    return x + y;
}
1
2
3
4

尽管您可能认为这会引起命名冲突，但此处并非如此。编译器能够根据函数调用中使用的参数来确定要调用的add()版本。实际上，只要每个add()函数都有唯一的参数，我们就可以定义任意数量的重载add()函数。

因此，还可以使用不同数量的参数来定义add()函数：

int add(int x, int y, int z)
{
    return x + y + z;
}
1
2
3
4

即使此add()函数具有3个参数而不是2个参数，因为这些参数与add()的任何其他版本都不相同，所以这是有效的。

函数返回类型不考虑唯一性。

重载函数时，不考虑函数的返回类型。因为我们确实能保证输入参数的唯一性，但是不能保证返回类型的唯一性。

考虑以下情况：要编写一个返回随机数的函数，但是需要一个返回int的版本和另一个返回double的版本。您可能会这样做：

int getRandomValue();
double getRandomValue();
1
2

编译器会将其标记为错误。这两个函数具有相同的参数（无），因此，第二个getRandomValue()将被视为第一个的错误重新声明。

解决此问题的最佳方法是为函数赋予不同的名称：

int getRandomInt();
double getRandomDouble();
1
2

另一种方法是使函数返回void，并将返回值作为out参数传递回调用方。

void getRandomValue(int &out);
void getRandomValue(double &out);
1
2

因为这些函数具有不同的参数，所以它们被认为是唯一的。但是，这样做有弊端。首先，语法很尴尬，您不能将此函数的输出直接输出到另一个函数的输入中。

// method 1: getRandomInt() returns an int
printValue(getRandomInt()); // easy
 
// method 2: getRandomValue() has an int out parameter
int temp; // now we need a temp variable
getRandomValue(temp); // to call getRandomValue(int) here
printValue(temp); // this has to be a separate line since getRandomValue() returns void
1

另外，传入的参数的类型必须与参数的类型完全匹配。由于这些原因，我们不建议您使用此方法。

Typedefs are not distinct

由于声明typedef不会引入新类型，因此以下两个Print（）声明被视为相同：

typedef char *string;
void print(string value);
void print(char *value);

How function calls are matched with overloaded functions

调用重载函数会导致以下三种可能的结果之一：

  • 找到一个匹配项。该调用被解析为特定的重载函数。
  • 找不到匹配项。参数不能与任何重载函数匹配。
  • 找到一个不明确的匹配项。参数匹配多个重载函数。

调用重载函数时，C ++将通过以下过程来确定将调用哪个版本的函数：

1)首先，C ++试图找到一个完全匹配的对象。在这种情况下，实际参数与重载函数之一的参数类型完全匹配。例如：

void print(char *value);
void print(int value);
 
print(0); // exact match with print(int)
1
2
3
4

尽管从技术上讲0可以匹配print(char *)（作为空指针），但它与print(int)完全匹配（匹配char *将需要隐式转换）。因此print(int)是可用的最佳匹配。

2)如果找不到完全匹配的内容，则C ++会尝试通过升级(promote)找到匹配的内容。

  • char，unsigned char, short提升为int.
  • Unsigned short可以提升为int或unsigned int，具体取决于int的大小.
  • Float is promoted to double.
  • Enum is promoted to int.

例如

void print(char *value);
void print(int value);
 
print('a'); // promoted to match print(int)
1
2
3
4

在这种情况下，因为没有print(char)，所以将char('a')提升为整数，然后将其与print(int)匹配。

3）如果无法升级，则C ++会尝试通过标准转换查找匹配项。标准转换包括：

  • 任何数字类型都将与任何其他数字类型匹配，包括无符号（例如int浮点数）.
  • 枚举将匹配数字类型的形式类型（例如，枚举到浮点数）.
  • 零将匹配指针类型和数字类型（例如0表示char *，0表示浮点数）.
  • 指针将与void指针匹配.
struct Employee; // defined somewhere else
void print(float value);
void print(Employee value);
 
print('a'); // 'a' converted to match print(float)
1
2
3
4
5

在这种情况下，因为没有print(char)（完全匹配），也没有print(int)（促销匹配），所以“ a”将转换为float并与print(float)匹配。

请注意，所有标准转换都被视为相等。没有标准转换被认为比其他任何转换都要好。

4)最后，C ++尝试通过用户定义的转换找到匹配项。尽管我们还没有介绍类，但是类（类似于struct）可以定义到其他类型的转换，这些转换可以隐式应用于该类的对象。例如，我们可以定义一个类X和一个用户定义的int转换。

class X; // with user-defined conversion to int
 
void print(float value);
void print(int value);
 
X value; // declare a variable named value of type class X
print(value); // value will be converted to an int and matched to print(int)

Ambiguous matches

如果每个重载函数都必须具有唯一的参数，那么调用怎么可能导致多个匹配？因为所有标准转换都被视为相等，并且所有用户定义的转换都被视为相等，所以如果函数调用通过标准转换或用户定义的转换与多个候选匹配，则将导致模棱两可的匹配。例如：

void print(unsigned int value);
void print(float value);
 
print('a');
print(0);
print(3.14159);
1
2
3
4
5
6

对于print('a')，C ++找不到精确匹配。它尝试将“ a”提升为int，但也没有print(int)。使用标准转换，它可以将“ a”转换为无符号的int和浮点值。因为所有标准转换都被视为相等，所以这是模棱两可的匹配。

print(0)相似。 0是一个int，并且没有print(int)。它通过标准转换匹配两个函数。

print(3.14159)可能有点令人惊讶，因为大多数程序员都认为它与print(float)匹配。但是请记住，所有字面量浮点值都为double，除非后缀为“ f”。 3.14159是双精度型，并且没有print(double)。因此，它通过标准转换匹配两个函数。

Ambiguous matches被视为编译时(compile time)错误。因此，在编译程序之前，需要消除歧义。有几种解决歧义匹配的方法：

  • 通常，最好的方法是简单地定义一个新的重载函数，该函数采用与您尝试使用该函数完全相同类型的参数。然后，C ++将能够找到函数调用的完全匹配项。

  • 或者，将模棱两可的参数显式转换为要调用的函数的类型。例如，要让print(0调用print(unsigned int)，您可以这样做：

    int x = 0;
print(static_cast<unsigned int>(x)); // will call print(unsigned int)
    1
    2

  • 如果您的参数是字面量，则可以使用文字后缀以确保您的文字被解释为正确的类型：

    print(0u); // will call print(unsigned int) since 'u' suffix is unsigned int
    1

Matching for functions with multiple arguments

如果有多个参数，则C ++会将匹配规则依次应用于每个参数。选择的函数是每个参数至少与所有其他函数匹配的函数，其中至少一个参数比所有其他函数匹配的更好。换句话说，对于至少一个参数，所选函数必须比所有其他候选函数提供更好的匹配，并且对于所有其他参数都必须提供更好的匹配。

在找到这种功能的情况下，无疑是最佳选择。如果找不到这样的函数，则该调用将被视为模棱两可（或不匹配）。

[[include]] <iostream>
 
void fcn(char c, int x)
{
	std::cout << 'a';
}
 
void fcn(char c, double x)
{
	std::cout << 'b';
}
 
void fcn(char c, float x)
{
	std::cout << 'c';
}
 
int main()
{
	fcn('x', 4);
}

Conclusion

函数重载可以显着降低程序的复杂度，同时几乎不会带来额外的风险。尽管这一课很长，而且看起来有些复杂（尤其是匹配规则），但实际上函数重载通常是透明的，没有任何问题。编译器将标记所有模棱两可的情况，通常可以轻松解决它们。
