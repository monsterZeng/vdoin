Passing arguments by reference

# Passing arguments by reference

​ 虽然按值传递在很多情况下都适用，但它有一些限制。首先，在向函数传递大型结构体或类时，传值将把实参的拷贝副本传递到函数形参中。在许多情况下，这是不必要的性能损失。第二，当按值传递参数时，向调用者返回值的唯一方法是通过函数的返回值。虽然这通常是合适的，但在有些情况下，让函数修改传入的参数会更清楚和有效。通过引用传递解决了这两个问题。

# Pass by reference

​ 要通过引用传递变量，只需将函数形参声明为引用而不是普通变量:

void addOne(int &ref) // ref is a reference variable
{
    ref = ref + 1;
}

​ 当函数被调用时，ref将成为实参的引用。由于对变量的引用与变量本身的处理完全相同，所以对引用所做的任何更改都会传递给实参!

​ 下面的例子演示了这一点:

[[include]]<iostream>
using namespace std;
void addOne(int &ref)
{
    ref = ref + 1;
}
 
int main()
{
    int value = 5;
 
    cout << "value = " << value << '\n'; 5
    addOne(value);
    cout << "value = " << value << '\n';6
    return 0;
}

Returning multiple values via out parameters

​ 有时我们需要一个函数返回多个值。但是，函数只能有一个返回值。返回多个值的一种方法是使用引用形参:

[[include]] <iostream>
[[include]] <cmath>    // for std==sin() and std==cos()
 
void getSinCos(double degrees, double &sinOut, double &cosOut)
{
    // sin() and cos() take radians, not degrees, so we need to convert
    static constexpr double pi { 3.14159265358979323846 }; // the value of pi
    double radians = degrees * pi / 180.0;
    sinOut = std::sin(radians);
    cosOut = std::cos(radians);
}
 
int main()
{
    double sin(0.0);
    double cos(0.0);
 
    // getSinCos will return the sin and cos in variables sin and cos
    getSinCos(30.0, sin, cos);
 
    std::cout << "The sin is " << sin << '\n';
    std::cout << "The cos is " << cos << '\n';
    return 0;
}

​ 这个函数接受一个参数(按值)作为输入，并“返回”两个参数(按引用)作为输出。仅用于向调用者返回值的参数被称为输出参数。我们用后缀“out”来命名这些out参数，表示它们是out参数。这有助于提醒调用者传递给这些参数的初始值并不重要，我们应该期待它们被重写。按照惯例，输出参数通常是最右边的参数。

​ 让我们更详细地探讨它是如何工作的。首先，main函数创建局部变量sin和cos。它们通过引用(而不是通过值)传递给函数getSinCos()。这意味着函数getSinCos()可以访问实际的sin和cos变量，而不仅仅是副本。getSinCos()相应地将新值赋给sin和cos(分别通过引用sinOut和cosOut)，这将覆盖sin和cos中的旧值。Main然后打印这些更新后的值。

​ 如果是通过值而不是引用传递sin和cos, 那么getSinCos()将更改sin和cos的副本，导致在函数结束时丢弃任何更改。但是，因为sin和cos是通过引用传递的，所以(通过引用)对sin或cos所做的任何更改都将在函数之外持久存在。因此，我们可以使用这种机制将值返回给调用者。

​ 这种方法虽然具有功能性，但有一些小缺点。首先，调用者必须传入参数来保存更新后的输出，即使它不打算使用它们。更重要的是，语法有点不自然，输入和输出参数都放在函数调用中。从调用者那端看，sin和cos不是参数，不会被改变。这可能是该方法最危险的部分(因为它可能导致错误)。一些程序员和公司认为这是一个足够大的问题，建议避免使用输出参数。

​ 就个人而言，我们建议尽可能避免使用其他参数。如果使用它们，使用“out”后缀(或前缀)命名参数(和输出参数)有助于清楚地表明值可能被修改。

Limitations of pass by reference

​ 非const引用只能引用非const的l值(例如，非const变量)，因此引用形参不能接受const l值或r值的实参(例如，字面量和表达式的结果)。

传递const引用

​ 如在简介中提到的，按值传递的主要缺点之一是，所有按值传递的参数都被复制到函数形参中。当参数是大型结构体或类时，这可能会花费大量时间。引用提供了一种避免这种惩罚的方法。通过引用传递实参时，将创建对实际实参的引用(这需要最少的时间)，并且不会复制值。这允许我们以最小的性能损失传递大型结构和类。

​ 然而，这也给我们带来了潜在的麻烦。当我们希望实参是只读的时，引用允许函数改变实参的值，这是不可取的。如果我们知道函数不应该改变实参的值，但又不想通过值传递，那么最好的解决方案是通过const引用传递。

​ const引用是一种不允许通过引用更改被引用的变量的引用。因此，如果使用const引用作为形参，就可以向调用者保证函数不会改变实参!

​ 下面的函数将产生一个编译错误:

void foo(const std::string &x) // x is a const reference
{
    x = "hello";  // compile error: a const reference cannot have its value changed!
}

使用const有以下几个原因:

  • 它要求编译器帮助确保不应该更改的值不被更改(如果您尝试更改，编译器将抛出一个错误)。
  • 它告诉程序员这个函数不会改变实参的值，这有助于调试。
  • const引用可以接受任何类型的实参，包括非Const l-values、Const l-values和r-values。

通过引用传递实参时，除非需要更改实参的值，否则始终使用const引用。

非const引用不能绑定到右值。带非const引用形参的函数不能用字面量或临时形参调用。

[[include]] <string>
 
void foo(std::string& text) {}
 
int main()
{
  std::string text{ "hello" };
  
  foo(text); // ok
  foo(text + " world"); // illegal, non-const references can't bind to r-values.
 
  return 0;
}

# References to pointers

可以通过引用传递指针，并让函数完全改变指针的地址:

[[include]] <iostream>
 
void foo(int *&ptr) // pass pointer by reference
{
	ptr = nullptr; // this changes the actual ptr argument passed in, not a copy
}
 
int main()
{
	int x = 5;
	int *ptr = &x;
	std::cout << "ptr is: " << (ptr ? "non-null" : "null") << '\n'; // prints non-null
	foo(ptr);
	std::cout << "ptr is: " << (ptr ? "non-null" : "null") << '\n'; // prints null
 
	return 0;
}

提醒一下，可以通过引用传递c风格数组。如果你需要函数改变数组的能力(例如排序函数)，或者你需要访问固定数组的数组类型信息(做sizeof()或for-each循环)，这是很有用的。然而，请注意，为了使其工作，你需要显式地在形参中定义数组大小:

[[include]] <iostream>
 
// Note: You need to specify the array size in the function declaration
void printElements(int (&arr)[4])
{
  int length{ sizeof(arr) / sizeof(arr[0]) }; // we can now do this since the array won't decay
  
  for (int i{ 0 }; i < length; ++i)
  {
    std::cout << arr[i] << '\n';
  }
}
 
int main()
{
    int arr[]{ 99, 20, 14, 80 };
    
    printElements(arr);
 
    return 0;
}

[[include]]<iostream>
template <int size>
void printElements(int (&arr)[size])
{
  int length{ sizeof(arr) / sizeof(arr[0]) }; // we can now do this since the array won't decay
  
  for (int i{ 0 }; i < length; ++i)
  {
    std::cout << arr[i] << '\n';
  }
}
int main(void){
  int arr[]{ 99, 20, 14, 80, 20, 30};
    
  printElements(arr);
  return 0;
}

引用传递的利弊

  • 利

    • 引用允许函数改变实参的值，这有时是有用的。否则，可以使用const引用来保证函数不会改变实参。
    • 由于没有创建实参的副本，因此按引用传递速度很快，即使在大型结构体或类中使用也是如此。
    • 引用可用于从函数返回多个值(通过out形参)。
    • 引用必须初始化，因此不必担心空值。

  • 弊

    • 因为非const引用不能用const l值或r值(例如文字或表达式)初始化，所以非const引用形参的实参必须是普通变量。
    • 很难区分由非const引用传递的实参是要作为输入或者输出，还是同时作为输入和输出。明智地使用const和命名后缀对我们的变量有帮助。
    • 从函数调用中无法判断实参是否会改变。按值传递的参数和按引用传递的参数看起来是一样的。我们只能通过函数声明来判断参数是通过值传递还是通过引用传递。这可能导致程序员没有意识到函数会改变实参的值的情况。

    什么时候使用引用传递

    • 传递结构或类时（如果为只读，则使用const）。
    • 当需要函数修改参数时。
    • 当您需要访问固定数组的类型信息时。

    什么时候不使用引用传递

    • 当传递不需要修改的基本类型时(使用按值传递)。

    对于结构、类和其他需要大量复制的类型，使用const引用传递代替值传递。
