Pointers and references to the base class of derived objects

# Pointers and references to the base class of derived objects

在构建派生类这一章中，您了解了当您创建一个派生类时，它由多个部分组成:一个部分用于每个继承类，另一个部分用于自身。

#include <string_view>
 
class Base
{
protected:
    int m_value;
 
public:
    Base(int value)
        : m_value{ value }
    {
    }
 
    std::string_view getName() const { return "Base"; }
    int getValue() const { return m_value; }
};
 
class Derived: public Base
{
public:
    Derived(int value)
        : Base{ value }
    {
    }
 
    std::string_view getName() const { return "Derived"; }
    int getValueDoubled() const { return m_value * 2; }
};

当我们创建一个派生类对象时，它包含一个基类部分(首先构造)和一个派生类部分(第二个构造)。记住，继承意味着两个类之间的is-a关系。由于基类是派生类一部分，派生类包含基类部分是合适的。

# Pointers, references, and derived classes

我们可以很直观地设置派生类指针和派生类对象的引用:

#include <iostream>
 
int main()
{
    Derived derived{ 5 };
    std::cout << "derived is a " << derived.getName() << " and has value " << derived.getValue() << '\n';
 
    Derived &rDerived{ derived };
    std::cout << "rDerived is a " << rDerived.getName() << " and has value " << rDerived.getValue() << '\n';
 
    Derived *pDerived{ &derived };
    std::cout << "pDerived is a " << pDerived->getName() << " and has value " << pDerived->getValue() << '\n';
 
    return 0;
}

然而，由于Derived有Base部分，一个更有趣的问题是c++是否允许我们设置Base指针或对Derived对象的引用。事实证明，我们可以!

#include <iostream>
 
int main()
{
    Derived derived{ 5 };
 
    // These are both legal!
    Base &rBase{ derived };
    Base *pBase{ &derived };
 
    std::cout << "derived is a " << derived.getName() << " and has value " << derived.getValue() << '\n';
    std::cout << "rBase is a " << rBase.getName() << " and has value " << rBase.getValue() << '\n';
    std::cout << "pBase is a " << pBase->getName() << " and has value " << pBase->getValue() << '\n';
 
    return 0;
}
/*
    derived is a Derived and has value 5
    rBase is a Base and has value 5
    pBase is a Base and has value 5
*/

这个结果可能不是你一开始所期望的!

事实证明，由于rBase和pBase是一个Base引用和指针，它们只能看到Base的成员(或Base继承的任何类)。所以即使Derived::getName()隐藏了派生对象的Base::getName()， Base指针/引用也不能看到Derived::getName()。因此，它们调用Base::getName()，这就是为什么rBase和pBase报告它们是Base而不是Derived。

请注意，这也意味着无法使用 rBase 或 pBase 调用 Derived::getValueDoubled()。他们无法在 Derived 中看到任何内容。

下面是另一个稍微复杂一点的例子，我们将在下一课中继续讨论:

#include <iostream>
#include <string_view>
#include <string>
 
class Animal
{
protected:
    std::string m_name;
 
    // We're making this constructor protected because
    // we don't want people creating Animal objects directly,
    // but we still want derived classes to be able to use it.
    Animal(std::string_view name)
        : m_name{ name }
    {
    }
    
    // To prevent slicing (covered later)
    Animal(const Animal&) = delete;
    Animal& operator=(const Animal&) = delete;
 
public:
    const std::string& getName() const { return m_name; }
    std::string_view speak() const { return "???"; }
};
 
class Cat: public Animal
{
public:
    Cat(std::string_view name)
        : Animal{ name }
    {
    }
 
    std::string_view speak() const { return "Meow"; }
};
 
class Dog: public Animal
{
public:
    Dog(std::string_view name)
        : Animal{ name }
    {
    }
 
    std::string_view speak() const { return "Woof"; }
};
 
int main()
{
    const Cat cat{ "Fred" };
    std::cout << "cat is named " << cat.getName() << ", and it says " << cat.speak() << '\n';
 
    const Dog dog{ "Garbo" };
    std::cout << "dog is named " << dog.getName() << ", and it says " << dog.speak() << '\n';
 
    const Animal *pAnimal{ &cat };
    std::cout << "pAnimal is named " << pAnimal->getName() << ", and it says " << pAnimal->speak() << '\n';
 
    pAnimal = &dog;
    std::cout << "pAnimal is named " << pAnimal->getName() << ", and it says " << pAnimal->speak() << '\n';
 
    return 0;
}
/*
    cat is named Fred, and it says Meow
    dog is named Garbo, and it says Woof
    pAnimal is named Fred, and it says ???
    pAnimal is named Garbo, and it says ???
*/

# Use for pointers and references to base classes

现在你可能会说，“上面的例子看起来有点傻。为什么我可以使用派生对象，还要设置一个指向派生对象基类的指针或引用? "事实证明，有相当多的理由。

首先，假设您想要编写一个函数来打印动物的名字和声音。如果不使用指向基类的指针，你就必须使用重载函数来编写它，就像这样:

void report(const Cat &cat)
{
    std::cout << cat.getName() << " says " << cat.speak() << '\n';
}
 
void report(const Dog &dog)
{
    std::cout << dog.getName() << " says " << dog.speak() << '\n';
}

不是很难，但是想想如果我们有30种不同的动物类型而不是2种会发生什么。你必须写出30个几乎相同的函数!另外，如果你添加了一种新的动物，你也需要为它写一个新的函数。考虑到唯一真正的区别是参数的类型，这是一个巨大的时间浪费。

然而，因为Cat和Dog来源于Animal，所以Cat和Dog有一个Animal的部分。因此，我们可以这样做是有意义的:

void report(const Animal &rAnimal)
{
    std::cout << rAnimal.getName() << " says " << rAnimal.speak() << '\n';
}

这将允许我们传入任何派生于Animal的类，甚至是我们在编写函数之后创建的类!我们得到的不是每个派生类一个函数，而是一个用于所有派生类的函数。

当然，问题在于，因为rAnimal是一个Animal引用，所以rAnimal.speak()将调用Animal::speak()，而不是speak()的派生版本。

其次，假设您有3只猫和3只狗，您希望将它们保存在一个数组中以便于访问。因为数组只能保存一种类型的对象，没有指向基类的指针或引用，所以必须为每种派生类型创建不同的数组，如下所示:

#include <array>
#include <iostream>
 
// Cat and Dog from the example above
 
int main()
{
    const auto cats{ std::to_array<Cat>({{ "Fred" }, { "Misty" }, { "Zeke" }}) };
    const auto dogs{ std::to_array<Dog>({{ "Garbo" }, { "Pooky" }, { "Truffle" }}) };
    
    // Before C++20
    // const std::array<Cat, 3> cats{{ { "Fred" }, { "Misty" }, { "Zeke" } }};
    // const std::array<Dog, 3> dogs{{ { "Garbo" }, { "Pooky" }, { "Truffle" } }};
 
    for (const auto& cat : cats)
    {
        std::cout << cat.getName() << " says " << cat.speak() << '\n';
    }
 
    for (const auto& dog : dogs)
    {
        std::cout << dog.getName() << " says " << dog.speak() << '\n';
    }
 
    return 0;
}

现在，考虑一下如果你有30种不同的动物会发生什么。你需要30个数组，每种动物一个!

然而，因为Cat和Dog都来源于Animal，所以我们可以这样做是有道理的:

#include <iostream>
 
int main()
{
    const Cat fred{ "Fred" };
    const Cat misty{ "Misty" };
    const Cat zeke{ "Zeke" };
 
    const Dog garbo{ "Garbo" };
    const Dog pooky{ "Pooky" };
    const Dog truffle{ "Truffle" };
 
    // Set up an array of pointers to animals, and set those pointers to our Cat and Dog objects
    const auto animals{ std::to_array<const Animal*>({&fred, &garbo, &misty, &pooky, &truffle, &zeke }) };
    
    // Before C++20, with the array size being explicitly specified
    // const std::array<const Animal*, 6> animals{ &fred, &garbo, &misty, &pooky, &truffle, &zeke };
    
    for (const auto animal : animals)
    {
        std::cout << animal->getName() << " says " << animal->speak() << '\n';
    }
 
    return 0;
}
