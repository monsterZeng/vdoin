The hidden “this” pointer

# The hidden “this” pointer

​ 面向对象程序员新手经常会问的一个关于类的问题是:“当一个成员函数被调用时，c++如何跟踪调用它的对象?”答案是c++利用了一个名为“this”的隐藏指针!让我们更详细地看看“this”。

​ 下面是一个简单的类，它保存一个整数，并提供构造函数和访问函数。注意，不需要析构函数，因为c++可以为我们清除整型成员变量。

class Simple
{
private:
    int m_id;
 
public:
    Simple(int id)
        : m_id{ id }
    {
    }
 
    void setID(int id) { m_id = id; }
    int getID() { return m_id; }
};
int main()
{
    Simple simple{1};
    simple.setID(2);
    std::cout << simple.getID() << '\n'; // answer is 2
 
    return 0;
}

​ 不知为何，当我们调用simple.setID(2);时，c++知道setID()函数应该对simple对象进行操作，而且m_id实际上引用的是simple.m_id。让我们来看看这背后的机制。

# *The hidden this pointer

​ 看看上面例子中的下面一行代码:

imple.setID(2);
1

​ 虽然对setID()函数的调用看起来只有一个参数，但实际上它有两个!编译时，编译器转换simple.setID(2);为以下:

setID(&simple, 2); // note that simple has been changed from an object prefix to a function argument!
1

​ 请注意，这现在只是一个标准函数调用，object simple(以前是一个对象前缀)现在通过address作为参数传递给函数。

​ 但这只是答案的一半。因为函数调用现在增加了一个实参，所以需要修改成员函数定义以接受(并使用)这个实参作为形参。因此，下面的成员函数:

  void setID(int id) { m_id = id; }   ->  void setID(Simple* const this, int id) { this->m_id = id; }
1

​ 当编译器编译一个普通成员函数时，它会隐式地给函数“this”添加一个新形参。this指针是一个隐藏的const指针，它保存了调用成员函数的对象的地址。

​ 还有一个细节要处理。在成员函数内部，任何类成员(函数和变量)也需要更新，以便它们引用成员函数被调用的对象。这很容易做到，通过添加“This ->”的前缀，每一个。因此，在函数setID()的函数体中，m_id(它是一个类成员变量)已经被转换为这个->m_id。因此，当“this”指向simple的地址时，this->m_id将解析为simple.m_id。

**note **

  • 当我们调用simple.setID(2)时，编译器实际上调用setID(&simple, 2)。
  • 在setID()中，this指针保存了simple对象的地址。
  • setID()中的任何成员变量都以" this-> "作为前缀。所以当我们说m_id = id时，编译器实际上是在执行this->m_id = id，在这种情况下更新simpe.m_id->id。

​ 好消息是，所有这些都是自动发生的，你是否记得它是如何工作的并不重要。您需要记住的是，所有普通成员函数都有一个“this”指针，指向调用函数的对象。

​ “this” always points to the object being operated on

​ 新程序员有时会对存在多少个“this”指针感到困惑。每个成员函数都有一个“this”指针参数，该参数设置为被操作对象的地址。考虑:

int main()
{
    Simple A{1}; // this = &A inside the Simple constructor
    Simple B{2}; // this = &B inside the Simple constructor
    A.setID(3); // this = &A inside member function setID
    B.setID(4); // this = &B inside member function setID
 
    return 0;
}

​ 请注意，“this”指针交替保存对象A或B的地址，这取决于我们是否在对象A或B上调用了成员函数。因为“this”只是一个函数形参，它不会给您的类增加任何内存使用量(只给成员函数调用，因为该形参需要传递给函数并存储在内存中)。

# Explicitly referencing “this”

​ 大多数时候，你不需要显式地引用“this”指针。然而，在一些情况下这样做是有用的:

​ 首先，如果构造函数(或成员函数)的形参与成员变量同名，可以使用" this "来消除歧义:

class Something
{
private:
    int data;
 
public:
    Something(int data)
    {
        this->data = data; // this->data is the member, data is the local parameter
    }
};

​ 注意，我们的构造函数接受与成员变量同名的形参。在本例中，“data”指的是参数，“this->data”指的是成员变量。虽然这是可以接受的编码实践，但我们发现在所有成员变量名上使用“m_*”前缀提供了一个更好的解决方案，它可以完全防止重复名称! 一些开发人员喜欢在所有类成员中显式添加this->。我们建议您避免这样做，因为这样做往往会降低代码的可读性，带来的好处也很少。使用m前缀是区分成员变量和非成员(局部)变量的一种更容易理解的方法.**

# Chaining member functions

​ 其次，有时让类成员函数返回它正在处理的对象作为返回值是很有用的。这样做的主要原因是允许一系列成员函数“链接”在一起，这样就可以在同一个对象上调用几个成员函数!你已经干了很长时间了。考虑一下这个常见的示例，您正在使用std::cout输出多个位的文本:

std::cout << "Hello, " << userName;
1

​ 在本例中，std::cout是一个对象，操作符<<是操作该对象的成员函数。编译器这样计算上面的代码片段:

(std::cout << "Hello, ") << userName;
1

​ 首先，操作符<<使用std::cout和字符串字面量“Hello”来将“Hello”打印到控制台。然而，由于这是表达式的一部分，操作符<<也需要返回一个值(或void)。如果操作符<<返回void，你会得到这样的结果:

(void) << userName;
1

​ 这显然没有任何意义(而且编译器会抛出一个错误)。相反，操作符<<返回*this，在这里是std::cout对象。这样，在计算了第一个操作符<<后，我们得到:

(std::cout) << userName;
1

​ 这样，我们只需要指定一次对象(在本例中是std::cout)，每个函数调用都会将其传递给下一个要使用的函数，从而允许我们将多个命令链接在一起。

​ 我们可以自己实施这种行为。考虑以下类:

class Calc
{
private:
    int m_value{0};
 
public:
 
    void add(int value) { m_value += value; }
    void sub(int value) { m_value -= value; }
    void mult(int value) { m_value *= value; }
 
    int getValue() { return m_value; }
};

//但是，如果我们让每个函数返回*this，我们就可以把调用链在一起。以下是带有“可链”函数的新版本的Calc:
class Calc
{
private:
    int m_value{};
 
public:
    Calc& add(int value) { m_value += value; return *this; }
    Calc& sub(int value) { m_value -= value; return *this; }
    Calc& mult(int value) { m_value *= value; return *this; }
 
    int getValue() { return m_value; }
};

[[include]] <iostream>
int main()

    calc.add(5).sub(3).mult(4);
 
    std::cout << calc.getValue() << '\n';
    return 0;

我们有效地将三行文字压缩成一个表达式!让我们仔细看看这是如何工作的。

首先，调用calc.add(5)，它将给我们的m_value添加5。然后add()返回*this，这只是对calc的引用，因此calc将是后续求值中使用的对象。下一个calc.sub(3)计算结果，从m_value中减去3，并再次返回calc。最后，calc.mult(4)将m_value乘以4并返回calc，以后不再使用，因此被忽略。因为每个函数在执行时都修改了calc，所以calc的m_value现在包含值(((0 + 5)- 3)* 4)
