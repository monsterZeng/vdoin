Basic inheritance in C++

# Basic inheritance in C++

既然我们已经讨论了抽象意义上的继承是什么，现在让我们讨论一下它在c++中是如何使用的。

c++中的继承发生在类之间。在继承(is-a)关系中，被继承的类称为父类、基类或超类，执行继承的类称为子类、派生类或子类。

在上图中，Fruit是父结点，Apple和Banana都是子结点。

在这个图中，三角形既是形状的子类，又是直角三角形的父类。

子类继承父类的行为(成员函数)和属性(成员变量)(受一些访问限制，我们将在未来的课程中讨论)。这些变量和函数成为派生类的成员。因为子类是更特殊的类，所以它们(当然)可以拥有特定于该类的自己的成员。稍后我们将看到一个这样的例子。

# A Person class

下面是一个简单的类来表示一个通用的人:

#include <string>
 
class Person
{
// In this example, we're making our members public for simplicity
public: 
    std::string m_name{};
    int m_age{};
 
    Person(const std::string& name = "", int age = 0)
        : m_name{ name }, m_age{ age }
    {
    }
 
    const std::string& getName() const { return m_name; }
    int getAge() const { return m_age; }
 
};

因为这个Person类被设计用来表示一个通用的人，所以我们只定义了对任何类型的人都通用的成员。每个人(无论性别、职业等)都有一个名字和年龄，所以这里代表了这些。

注意，在本例中，我们将所有变量和函数设置为公共。这纯粹是为了让这些示例保持简单。通常我们将变量设置为私有。我们将在本章后面讨论访问控制以及它们如何与继承交互。

# A BaseballPlayer class

假设我们想要编写一个程序来跟踪一些棒球运动员的信息。棒球运动员需要包含特定于棒球运动员的信息——例如，我们可能希望存储球员的击球率和他们击出的本垒打数。

这是我们不完整的Baseball player类:

class BaseballPlayer
{
// In this example, we're making our members public for simplicity
public:
    double m_battingAverage{};
    int m_homeRuns{};
 
    BaseballPlayer(double battingAverage = 0.0, int homeRuns = 0)
       : m_battingAverage{battingAverage}, m_homeRuns{homeRuns}
    {
    }
};

现在，我们还想跟踪棒球运动员的姓名和年龄，并且我们已经将这些信息作为Person类的一部分。

我们有三个选择如何添加姓名和年龄的棒球运动员:

  1. 将名字和年龄直接作为成员添加到BaseballPlayer类中。这可能是最糟糕的选择，因为我们正在复制Person类中已经存在的代码。任何对Person的更新也必须在BaseballPlayer中进行。
  2. 使用组合将Person添加为BaseballPlayer的成员。但我们必须问自己，一个棒球运动员有一个人吗?不,它不是。所以这不是正确的范例。
  3. 让BaseballPlayer从Person继承这些属性。记住，继承代表的是一种is-a关系。棒球运动员也是人吗?是的,它是。所以继承是一个很好的选择。

# Making BaseballPlayer a derived class

要让BaseballPlayer继承我们的Person类，语法相当简单。在类BaseballPlayer声明之后，我们使用冒号、单词public和希望继承的类名。这叫做公共继承。我们将在未来的课程中更多地讨论公共继承的含义。

// BaseballPlayer publicly inheriting Person
class BaseballPlayer : public Person
{
public:
    double m_battingAverage{};
    int m_homeRuns{};
 
    BaseballPlayer(double battingAverage = 0.0, int homeRuns = 0)
       : m_battingAverage{battingAverage}, m_homeRuns{homeRuns}
    {
    }
};

使用派生图，我们的继承就像这样:

当BaseballPlayer从Person继承时，BaseballPlayer从Person获取成员函数和变量。此外，BaseballPlayer定义了自己的两个成员:m_battingAverage和m_homeRuns。这是有意义的，因为这些属性是特定于一个BaseballPlayer，而不是任何Person。

因此，BaseballPlayer对象将有4个成员变量:m_battingAverage和m_homeRuns来自BaseballPlayer,m_name和m_age来自Person。

#include <iostream>
#include <string>
 
class Person
{
public:
    std::string m_name{};
    int m_age{};
 
    Person(const std::string& name = "", int age = 0)
        : m_name{name}, m_age{age}
    {
    }
 
    const std::string& getName() const { return m_name; }
    int getAge() const { return m_age; }
 
};
 
// BaseballPlayer publicly inheriting Person
class BaseballPlayer : public Person
{
public:
    double m_battingAverage{};
    int m_homeRuns{};
 
    BaseballPlayer(double battingAverage = 0.0, int homeRuns = 0)
       : m_battingAverage{battingAverage}, m_homeRuns{homeRuns}
    {
    }
};
 
int main()
{
    // Create a new BaseballPlayer object
    BaseballPlayer joe{};
    // Assign it a name (we can do this directly because m_name is public)
    joe.m_name = "Joe";
    // Print out the name
    std::cout << joe.getName() << '\n'; // use the getName() function we've acquired from the Person base class
 
    return 0;
}

# An Employee derived class

现在让我们编写另一个同样继承自Person的类。这一次，我们将编写一个Employee类。雇员“是”人，所以使用继承是合适的:

// Employee publicly inherits from Person
class Employee: public Person
{
public:
    double m_hourlySalary{};
    long m_employeeID{};
 
    Employee(double hourlySalary = 0.0, long employeeID = 0)
        : m_hourlySalary{hourlySalary}, m_employeeID{employeeID}
    {
    }
 
    void printNameAndSalary() const
    {
        std::cout << m_name << ": " << m_hourlySalary << '\n';
    }
};

#include <iostream>
#include <string>
 
class Person
{
public:
    std::string m_name{};
    int m_age{};
 
    const std::string& getName() const { return m_name; }
    int getAge() const { return m_age; }
 
    Person(const std::string& name = "", int age = 0)
        : m_name{name}, m_age{age}
    {
    }
};
 
// Employee publicly inherits from Person
class Employee: public Person
{
public:
    double m_hourlySalary{};
    long m_employeeID{};
 
    Employee(double hourlySalary = 0.0, long employeeID = 0)
        : m_hourlySalary{hourlySalary}, m_employeeID{employeeID}
    {
    }
 
    void printNameAndSalary() const
    {
        std::cout << m_name << ": " << m_hourlySalary << '\n';
    }
};
 
int main()
{
    Employee frank{20.25, 12345};
    frank.m_name = "Frank"; // we can do this because m_name is public
 
    frank.printNameAndSalary();// Frank: 20.25
    
    return 0;
}

# Inheritance chains

类也可以去继承派生类。这样做没有什么值得注意或特别的——一切都像上面的例子一样进行。

例如，让我们编写一个Supervisor类。一个主管是一个雇员，也就是一个人。我们已经写了一个Employee类，所以让我们用它作为派生Supervisor的基类:

class Supervisor: public Employee
{
public:
    // This Supervisor can oversee a max of 5 employees
    long m_overseesIDs[5]{};
};

所有的Supervisor对象都继承了Employee和Person的函数和变量，并添加了自己的m_overseesid成员变量。

通过构建这样的继承链，我们可以创建一组非常通用(在顶部)的可重用类，并在继承的每个级别上逐渐变得更加具体。

# Why is this kind of inheritance useful?

继承基类意味着我们不必在派生类中重新定义基类的信息。通过继承自动接收基类的成员函数和成员变量，然后简单地添加所需的附加函数或成员变量。这不仅节省了工作，而且还意味着如果我们更新或修改基类(例如添加新函数，或修复一个bug)，所有的派生类都会自动继承这些更改!

例如，如果我们向Person添加了一个新函数，那么Employee和Supervisor都会自动获得对它的访问权。如果我们向Employee添加一个新变量，Supervisor也可以访问它。这允许我们以一种简单、直观和低维护的方式构建新类!

# Conclusion

继承允许我们通过让其他类继承它们的成员来重用类。在未来的课程中，我们将继续探索这是如何工作的。
