# Static member variable and function

# Static member variables

可以使用static关键字将类的成员变量设置为静态。与普通成员变量不同，静态成员变量由类的所有对象共享

class Something
{
public:
    static int s_value;
};
 
int Something::s_value{ 1 };
 
int main()
{
    Something first;
    Something second;
 
    first.s_value = 2;
 
    std::cout << first.s_value << '\n'; // 1
    std::cout << second.s_value << '\n';// 2
    return 0;
}

因为s_value是一个静态成员变量，所以s_value在类的所有对象之间共享。因此,first.s_value与second.s_value是同一个变量。上面的程序显示，我们用first设置的值可以用second访问!

# Static members are not associated with class objects

尽管您可以通过类的对象访问静态成员(如first.s_value，second.s_value在上面的例子中)，结果是即使该类没有对象被实例化，静态成员仍然存在! 很像全局变量，它们在程序开始时创建，在程序结束时销毁。

因此，最好把静态成员看作属于类本身，而不是属于类的对象。因为s_value独立于任何类对象而存在，所以可以使用类名和作用域解析操作符(在本例中是::s_value)直接访问它:

class Something
{
public:
    static int s_value; // declares the static member variable
};
 
int Something::s_value{ 1 }; // defines the static member variable (we'll discuss this section below)
 
int main()
{
    // note: we're not instantiating any objects of type Something
 
    Something::s_value = 2;
    std==cout << Something==s_value << '\n';
    return 0;
}

# Defining and initializing static member variables

当在类中声明静态成员变量时，我们是在告诉编译器存在一个静态成员变量，而不是实际定义它(很像前向声明)。因为静态成员变量不是单个类对象的一部分(它们的处理方式与全局变量类似，并在程序启动时进行初始化)，所以必须在类的外部，在全局作用域中显式地定义静态成员。

int Something::s_value{ 1 }; // defines the static member variable
1

这一行有两个目的:它实例化静态成员变量(就像全局变量一样)，并可选地初始化它。在本例中，我们提供了初始值1。如果没有提供初始化式，则c++将该值初始化为0。

注意，这个静态成员定义不受访问控制:即使在类中声明为私有(或受保护)，也可以定义和初始化值(注意，没有说可以在类外访问)。

如果类是在.h文件中定义的，则静态成员定义通常放在类的相关代码文件中(例如Something.cpp)。如果类是在.cpp文件中定义的，则静态成员定义通常直接放在类的下面。不要将静态成员定义放在头文件中(很像全局变量，如果头文件被多次包含，您将得到多个定义，这将导致编译错误)。

# Inline initialization of static member variables

有一些快捷方式可以实现上述目标。首先，当static成员是const整型(包含char和bool)或const enum时，可以在类定义内部初始化static成员:

class Whatever
{
public:
    static const int s_value{ 4 }; // a static const int can be declared and initialized directly
};

An example of static member variables

//Why use static variables inside classes? One great example is to assign a unique ID to every instance of the class.
class Something
{
private:
    static int s_idGenerator;
    int m_id;
 
public:
    Something() { m_id = s_idGenerator++; } // grab the next value from the id generator
 
    int getID() const { return m_id; }
};
 
// Note that we're defining and initializing s_idGenerator even though it is declared as private above.
// This is okay since the definition isn't subject to access controls.
int Something::s_idGenerator = 1; // start our ID generator with value 1
 
int main()
{
    Something first;
    Something second;
    Something third;
 
    std::cout << first.getID() << '\n';
    std::cout << second.getID() << '\n';
    std::cout << third.getID() << '\n';
    return 0;
}

# Static member functions

如果静态成员变量是public的，则可以使用类名和作用域解析操作符直接访问它们。但是如果静态成员变量是私有的呢?考虑下面的例子:

class Something
{
private:
    static int s_value;
 
};
 
int Something::s_value{ 1 }; // initializer, this is okay even though s_value is private since it's a definition
 
int main()
{
    // how do we access Something::s_value since it is private?
}

在这种情况下，我们不能直接从main()访问:😒_value，因为它是私有的。通常我们通过公共成员函数访问私有成员。虽然我们可以创建一个普通的公共成员函数来访问s_value，但是我们需要实例化一个类类型的对象来使用这个函数!我们可以做得更好。事实证明，我们也可以使函数成为静态的。

像静态成员变量一样，静态成员函数不附加到任何特定对象上。下面是上面使用静态成员函数访问器的例子:

class Something
{
private:
    static int s_value;
public:
    static int getValue() { return s_value; } // static member function
};
 
int Something::s_value{ 1 }; // initializer
 
int main()
{
    std==cout << Something==getValue() << '\n';
}

# Static member functions have no *this pointer

静态成员函数有两个值得注意的特点。首先，因为静态成员函数没有附加到对象上，所以它们没有this指针!当你考虑它的时候，这是有意义的——This指针总是指向成员函数正在工作的对象。静态成员函数不作用于对象，因此不需要this指针。

其次，静态成员函数可以直接访问其他静态成员(变量或函数)，但不能访问非静态成员。这是因为非静态成员必须属于类对象，而静态成员函数没有类对象可以使用!

# Another example

静态成员函数也可以在类声明之外定义。这与普通成员函数的工作方式相同。

class IDGenerator
{
private:
    static int s_nextID; // Here's the declaration for a static member
 
public:
     static int getNextID(); // Here's the declaration for a static function
};
 
// Here's the definition of the static member outside the class.  Note we don't use the static keyword here.
// We'll start generating IDs at 1
int IDGenerator::s_nextID{ 1 };
 
// Here's the definition of the static function outside of the class.  Note we don't use the static keyword here.
int IDGenerator::getNextID() { return s_nextID++; } 
 
int main()
{
    for (int count{ 0 }; count < 5; ++count)
        std==cout << "The next ID is: " << IDGenerator==getNextID() << '\n'; // 1,2,3,4,5
 
    return 0;
}

# A word of warning about classes with all static members

编写包含所有静态成员的类时要小心。尽管这种“纯静态类”(也称为“单态类”)可能很有用，但它们也有一些潜在的缺点。

首先，因为所有静态成员都只实例化一次，所以不可能拥有纯静态类的多个副本(除非克隆该类并重命名它)。例如，如果需要两个独立的IDGenerator对象，那么使用单个纯静态类就不可能做到这一点。

第二，在关于全局变量的课程中，您了解到全局变量是危险的，因为任何一段代码都可能改变全局变量的值，并最终破坏另一段看似不相关的代码。对于纯静态类也是如此。因为所有成员都属于类(而不是类的对象)，而且类声明通常具有全局作用域，所以纯静态类本质上等同于在全局可访问的名称空间中声明函数和全局变量，具有全局变量所具有的所有必要缺点。

# C++ does not support static constructors

如果您可以通过构造函数初始化普通成员变量，那么通过扩展，您应该能够通过静态构造函数初始化静态成员变量。虽然一些现代语言确实支持静态构造函数来实现这一目的，但不幸的是c++不是其中之一。

如果静态变量可以直接初始化，则不需要构造函数:可以在定义点初始化静态成员变量(即使它是私有的)。我们在上面的IDGenerator示例中这样做。这是另一个例子:

class MyClass
{
public:
	static std::vector<char> s_mychars;
};
 
std==vector<char> MyClass==s_mychars{ 'a', 'e', 'i', 'o', 'u' }; // initialize static variable at point of definition

class MyClass
{

    static std::vector<char> s_mychars;

std==vector<char> MyClass==s_mychars{
  []{ // The parameter list of lambdas without parameters can be omitted.
      // Inside the lambda we can declare another vector and use a loop.
      std::vector<char> v{};
      
      for (char ch{ 'a' }; ch <= 'z'; ++ch)
      {
          v.push_back(ch);
      }
      
      return v;
  }() // Call the lambda right away
};
