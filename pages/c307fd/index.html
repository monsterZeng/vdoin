Virtual destructors, virtual assignment, and overriding virtualization

# Virtual destructors, virtual assignment, and overriding virtualization

# Virtual destructors

尽管c++为您的类提供了默认析构函数(如果您自己不提供)，但有时您会希望提供自己的析构函数(特别是当类需要释放内存时)。如果要处理继承，应该始终将析构函数设为虚函数。考虑以下例子:

#include <iostream>
class Base
{
public:
    ~Base() // note: not virtual
    {
        std::cout << "Calling ~Base()\n";
    }
};
 
class Derived: public Base
{
private:
    int* m_array;
 
public:
    Derived(int length)
      : m_array{ new int[length] }
    {
    }
 
    ~Derived() // note: not virtual (your compiler may warn you about this)
    {
        std::cout << "Calling ~Derived()\n";
        delete[] m_array;
    }
};
 
int main()
{
    Derived *derived { new Derived(5) };
    Base *base { derived };
 
    delete base;
 
    return 0;
}
//Calling ~Base()

因为base是一个base指针，所以当删除base时，程序会查看base析构函数是否为虚函数。它不是，所以它假设它只需要调用Base析构函数。我们可以从上面的例子中看到这一点:

但是，我们真的希望delete函数调用Derived的析构函数(该析构函数将依次调用Base的析构函数)，否则m_array将不会被删除。为此，我们将Base的析构函数设为虚函数:

#include <iostream>
class Base
{
public:
    virtual ~Base() // note: virtual
    {
        std::cout << "Calling ~Base()\n";
    }
};
 
class Derived: public Base
{
private:
    int* m_array;
 
public:
    Derived(int length)
      : m_array{ new int[length] }
    {
    }
 
    virtual ~Derived() // note: virtual
    {
        std::cout << "Calling ~Derived()\n";
        delete[] m_array;
    }
};
 
int main()
{
    Derived *derived { new Derived(5) };
    Base *base { derived };
 
    delete base;
 
    return 0;
}
//Calling ~Derived()
//Calling ~Base()

Rule:

Whenever you are dealing with inheritance, you should make any explicit destructors virtual.

与普通虚成员函数一样，如果基类函数为虚函数，则所有派生重写都将被视为虚函数，而不管它们是否被指定为虚函数。没有必要为了将其标记为虚函数而创建一个空的派生类析构函数。

# Virtual assignment

赋值操作符可以是虚的。但是，不像在析构函数中，虚拟化总是一个好主意，对赋值操作符进行虚拟化实际上会带来很多麻烦，并涉及本教程范围之外的一些高级主题。因此，为了简单起见，我们建议你们暂时不要把赋值操作符变成虚拟的。

# Ignoring virtualization

您可能很少想要忽略函数的虚拟化。例如，考虑以下代码:

class Base
{
public:
    virtual const char* getName() const { return "Base"; }
};
 
class Derived: public Base
{
public:
    virtual const char* getName() const { return "Derived"; }
};

可能在某些情况下，你想要一个指向派生对象的Base指针来调用Base::getName()而不是Derived::getName()。要做到这一点，只需使用范围解析操作符:

#include <iostream>
int main()
{
    Derived derived;
    const Base &base { derived };
    // Calls Base::GetName() instead of the virtualized Derived::GetName()
    std::cout << base.Base::getName() << '\n';
 
    return 0;
}

# Should we make all destructors virtual?

这是新程序员经常问的一个问题。正如上面的示例所指出的，如果基类析构函数没有标记为虚函数，那么如果程序员稍后删除指向派生对象的基类指针，程序就有内存泄漏的风险。避免这种情况的一种方法是将所有析构函数标记为虚函数。但是你应该吗?

很容易说yes，这样您以后就可以使用任何类作为基类——但是这样做会有性能损失(向类的每个实例添加一个虚拟指针)。所以你必须平衡成本和你的意图。

传统的智慧(最初由著名的c++专家Herb Sutter提出)建议避免非虚析构函数内存泄漏的情况如下:“基类的析构函数应该是公共的和虚的，或者是受保护的和非虚的。”具有受保护析构函数的类不能通过指针删除，从而防止在基类具有非虚析构函数时通过基指针意外删除派生类。不幸的是，这也意味着不能通过基类指针删除基类，这本质上意味着不能删除基类

现在计算机语言已经引入了final说明符，我们的建议如下:

  • 如果你想继承你的类，请确保析构函数是虚的。
  • 如果您不打算继承您的类，请将您的类标记为final。这将首先防止其他类从它继承，而不会对类本身施加任何其他使用限制。
