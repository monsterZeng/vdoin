Virtual functions and polymorphism

# Virtual functions and polymorphism

在上一节关于派生对象基类的指针和引用的课中，我们看了一些示例，在这些示例中，使用基类的指针或引用可以简化代码。然而，在每一种情况下，我们都会遇到这样的问题:基指针或引用只能调用函数的基版本，而不能调用派生版本。

#include <iostream>
#include <string_view>
 
class Base
{
public:
    std::string_view getName() const { return "Base"; }
};
 
class Derived: public Base
{
public:
    std::string_view getName() const { return "Derived"; }
};
 
int main()
{
    Derived derived;
    Base &rBase{ derived };
    std::cout << "rBase is a " << rBase.getName() << '\n';//rBase is a Base
 
    return 0;
}

因为rBase是一个Base引用，所以它调用Base::getName()，即使它实际上引用的是派生对象的Base部分。

在这节课中，我们将展示如何使用虚函数来解决这个问题。

# Virtual functions and polymorphism

虚函数是一种特殊类型的函数，当调用该函数时，它将解析为存在于基类和派生类之间的函数的最派生版本。这种能力称为多态。如果派生函数具有与基版本相同的签名(名称、形参类型以及是否为const)和返回类型，则认为它是匹配的。这样的函数称为覆盖。

要使函数为虚函数，只需将" virtual "关键字放在函数声明之前。

下面是一个虚函数的例子:

#include <iostream>
#include <string_view>
 
class Base
{
public:
    virtual std::string_view getName() const { return "Base"; } // note addition of virtual keyword
};
 
class Derived: public Base
{
public:
    virtual std::string_view getName() const { return "Derived"; }
};
 
int main()
{
    Derived derived;
    Base &rBase{ derived };
    std::cout << "rBase is a " << rBase.getName() << '\n';//rBase is a Derived
 
    return 0;
}

因为rBase是派生对象的Base部分的引用，所以当计算rBase.getName()时，它通常会解析为Base::getName()。但是，Base::getName()是虚函数，它告诉程序去查看在Base和Derived之间是否有更多派生版本的函数。在本例中，它将解析为Derived::getName()!

    C c;
    A &rBase{ c };

你认为这个程序将输出什么?

让我们看看这是如何工作的。首先，我们实例化一个C类对象。rBase是一个A引用，我们设置它来引用C对象的A部分。最后，我们调用rBase.getName()。rBase.getName()的计算结果是A::getName()。但是，A::getName()是虚函数的，因此编译器将调用A和C之间派生的最基本的匹配。在本例中，即C::getName()。注意，它不会调用D::getName()，因为我们的原始对象是C，而不是D，所以只考虑a和C之间的函数。

当Animal .speak()被求值时，程序注意到Animal::speak()是一个虚函数。在animal引用Cat对象的animal部分的情况下，程序会查看animal和Cat之间的所有类，看看是否能找到更派生的函数。在这种情况下，它会找到Cat::speak()。在animal引用Dog对象的animal部分的情况下，程序将函数调用解析为Dog::speak()。

注意，我们没有将Animal::getName()设为虚函数。这是因为getName()在任何派生类中都不会被重写，因此没有必要。

类似地，下面的数组示例现在按预期工作:

Cat fred{ "Fred" };
Cat misty{ "Misty" };
Cat zeke{ "Zeke" };
 
Dog garbo{ "Garbo" };
Dog pooky{ "Pooky" };
Dog truffle{ "Truffle" };
 
// Set up an array of pointers to animals, and set those pointers to our Cat and Dog objects
Animal *animals[]{ &fred, &garbo, &misty, &pooky, &truffle, &zeke };
 
for (const auto *animal : animals)
    std::cout << animal->getName() << " says " << animal->speak() << '\n';
/*

Garbo says Woof
Misty says Meow
Pooky says Woof
Truffle says Woof
Zeke says Meow

即使这两个例子只使用Cat和Dog，我们从Animal派生的任何其他类也可以使用我们的report()函数和Animal数组，而无需进一步修改!这可能是虚函数的最大好处——能够以这样一种方式构造您的代码，使新派生的类无需修改就能自动使用旧代码!

警告:要使用派生类函数，派生类函数的签名必须与基类虚函数的签名精确匹配。如果派生类函数具有不同的形参类型，则程序可能仍能正常编译，但虚函数将不能按预期解析。

# Use of the virtual keyword

如果一个函数被标记为虚函数，那么所有匹配的覆盖也被认为是虚函数，即使它们没有被显式标记为虚函数。但是，在派生函数上使用关键字virtual并没有什么坏处，它提醒我们该函数是虚函数而不是普通函数。因此，对派生类中的虚拟函数使用virtual关键字通常是一个好主意，即使它不是严格必要的。

# Return types of virtual functions

一般情况下，虚函数的返回类型必须与其覆盖类型匹配。考虑以下例子:

class Base
{
public:
    virtual int getValue() const { return 5; }
};
 
class Derived: public Base
{
public:
    virtual double getValue() const { return 6.78; }
};

# Do not call virtual functions from constructors or destructors

这里有另一个经常让毫无戒心的新程序员犯难的陷阱。不应该从构造函数或析构函数中调用虚函数。为什么?

记住，在创建派生类时，首先构造基类部分。如果要从基类构造函数调用虚函数，而类的派生部分甚至还没有创建，它将无法调用函数的派生版本，因为派生函数没有派生对象可处理。在c++中，它将调用Base版本。

析构函数也存在类似的问题。如果在基类析构函数中调用虚函数，它将始终解析为该函数的基类版本，因为类的派生部分将已经被撤销。

Rule

Never call virtual functions from constructors or destructors.

# The downside of virtual functions

因为大多数时候你希望你的函数是虚函数的，为什么不让所有的函数都是虚的呢?答案是它的效率很低——解析一个虚函数调用比解析一个常规函数调用花费的时间更长。此外，编译器还必须为每个具有一个或多个虚函数的类对象分配一个额外的指针。我们将在本章的未来课程中进一步讨论这个问题。
