# Unsupervised pre-training

GPT使用的是Transformer的decoder。

给定一个无监督的序列语料库U=u1,⋯,un\mathcal{U}={u_1},\cdots,u_nU=u​1​​,⋯,u​n​​，使用标准的语言模型目标去最大化下列的概率

L1(U)=∑ilogP(ui∣ui−k,...,ui−1;Θ)L_1(\mathcal{U})=\sum_{i}\log P(u_i|u_{i-k},...,u_{i-1};\Theta) L​1​​(U)=​i​∑​​logP(u​i​​∣u​i−k​​,...,u​i−1​​;Θ)

这里，k是语境窗口的大小，只token uiu_iu​i​​的前k个词。这个使用参数Θ\ThetaΘ的神经网络去建模条件概率PPP，这个参数使用随机梯度下降去训练。

# Supervised fine-tuning

在使用上述目标训练完模型时，将参数应用到一个监督目标任务上。假设一个有标签的数据集C\mathcal{C}C，每一个例子都包含一个token的序列x1,x2,...,xmx^1,x^2,...,x^mx​1​​,x​2​​,...,x​m​​ 和一个标签yyy​。将token序列输入到预训练模型中去获得final transformer block's activation hlmh_l^mh​l​m​​，将其输入到一个额外的线性输出层去预测标签

P(y∣x1,x2,...,xm)=softmax(hlmWy)P(y|x^1,x^2,...,x^m)=\text{softmax}(h_l^mW_y) P(y∣x​1​​,x​2​​,...,x​m​​)=softmax(h​l​m​​W​y​​)

需要最大化的目标函数为：

L2(C)=∑(x,y)logP(y∣x1,...,xm)L_2(\mathcal{C})=\sum_{(x,y)}\log{P(y|x^1,...,x^m)} L​2​​(C)=​(x,y)​∑​​logP(y∣x​1​​,...,x​m​​)

我们还发现，将语言建模作为微调的辅助目标有助于(a)改进监督模型的泛化，(b)加速收敛。所以，最终的优化目标为

L3(C)=L2(C)+λL1(C)L_3(\mathcal{C})=L_2(\mathcal{C})+\lambda L_1(\mathcal{C}) L​3​​(C)=L​2​​(C)+λL​1​​(C)

也就是说，在微调时增加预训练的任务有助于上述两个改进。

# GPT 2.0

GPT2.0模型与GPT1.0模型类型，数据量大大增加。在结构上的修改为

  • 将Layer normalization移动到每个block的输入，类似于预激活残差网络。
  • 在最后的self - attention块之后添加了一个额外的Layer normalization。
  • 采用了修正的初始化方法，该方法考虑了残差路径随模型深度的累积。我们在初始化时将residual layer的权重按1/√N进行缩放，其中N为第N层residual layer。
