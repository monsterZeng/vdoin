The override and final specifiers, and covariant return types

# The override and final specifiers, and covariant return types

为了解决继承的一些常见问题，c++ 11在c++中添加了两个特殊标识符:override和final。注意，这些标识符不被认为是关键字——它们是在特定上下文中具有特殊含义的普通标识符。

虽然final并不常用，但override是一个非常棒的附加功能，您应该经常使用它。在这节课中，我们将看一看这两者，以及虚函数覆盖返回类型必须匹配的规则的一个例外。

# The override specifier

正如我们在前一课中提到的，只有在签名和返回类型完全匹配时，派生类虚函数才被认为是重写。这可能会导致意外的问题，本来打算重写的函数实际上不是。

考虑下面例子：

class A
{
public:
	virtual const char* getName1(int x) { return "A"; }
	virtual const char* getName2(int x) { return "A"; }
};
 
class B : public A
{
public:
	virtual const char* getName1(short int x) { return "B"; } // note: parameter is a short int
	virtual const char* getName2(int x) const { return "B"; } // note: function is const
};
 
int main()
{
	B b{};
	A& rBase{ b };
	std::cout << rBase.getName1(1) << '\n';//A
	std::cout << rBase.getName2(2) << '\n';//A
 
	return 0;
}

因为rBase是A对B对象的引用，所以这里的意图是使用虚函数来访问B::getName1()和B::getName2()。然而，因为B::getName1()接受了一个不同的形参(一个short int而不是一个int)，它不被认为是a::getName1()的重写。更隐蔽的是，因为B::getName2()是const，而A::getName2()不是，所以B::getName2()不被认为是A::getName2()的重写。

在这个特殊的例子中，因为A和B只打印它们的名字，很容易看出我们弄错了重写，调用了错误的虚函数。然而，在更复杂的程序中，函数的行为或返回值没有打印出来，这样的问题很难调试。

为了帮助解决应该被覆盖但没有被覆盖的函数的问题，c++ 11引入了override说明符。override说明符可以应用于任何override函数，方法是将该说明符放在const所在的位置。如果函数没有覆盖基类函数(或应用于非虚函数)，编译器将把该函数标记为错误。

class A
{
public:
	virtual const char* getName1(int x) { return "A"; }
	virtual const char* getName2(int x) { return "A"; }
	virtual const char* getName3(int x) { return "A"; }
};
 
class B : public A
{
public:
	virtual const char* getName1(short int x) override { return "B"; } // compile error, function is not an override
	virtual const char* getName2(int x) const override { return "B"; } // compile error, function is not an override
	virtual const char* getName3(int x) override { return "B"; } // okay, function is an override of A::getName3(int)
 
};
 
int main()
{
	return 0;
}

上面的程序产生两个编译错误:一个是针对B::getName1()的，另一个是针对B::getName2()的，因为它们都不重写先前的函数。B::getName3()确实覆盖了A::getName3()，所以这一行不会产生错误。

使用override说明符不会造成性能损失，而且它有助于避免无意的错误。因此，我们强烈建议在重写每个虚函数时使用它，以确保您实际上已经重写了您认为您拥有的函数。

Apply the override specifier to every intended override function you write.

# The final specifier

在某些情况下，您可能不希望有人能够重写虚函数，或从类继承。final说明符可以用来告诉编译器强制执行此操作。如果用户试图重写final函数或从被指定为final的类继承，编译器将给出编译错误。

在我们想要限制用户重写一个函数的情况下，final说明符被使用在覆盖说明符所在的相同位置，如下所示:

class A
{
public:
	virtual const char* getName() { return "A"; }
};
 
class B : public A
{
public:
	// note use of final specifier on following line -- that makes this function no longer overridable
	virtual const char* getName() override final { return "B"; } // okay, overrides A::getName()
};
 
class C : public B
{
public:
	virtual const char* getName() override { return "C"; } // compile error: overrides B::getName(), which is final
};

在上面的代码中，B::getName()覆盖了A::getName()，这很好。但是B::getName()具有最终的说明符，这意味着对该函数的任何进一步重写都应视为错误。实际上，C::getName()尝试重写B::getName()(重写说明符在这里并不相关，它只是为了良好的实践)，因此编译器将给出一个编译错误。

在我们想要防止从类继承的情况下，最终的说明符应用在类名之后:

class A
{
public:
	virtual const char* getName() { return "A"; }
};
 
class B final : public A // note use of final specifier here
{
public:
	virtual const char* getName() override { return "B"; }
};
 
class C : public B // compile error: cannot inherit from final class
{
public:
	virtual const char* getName() override { return "C"; }
};

# Covariant return types

有一种特殊情况，派生类虚函数覆盖的返回类型可能与基类不同，但仍然被认为是匹配的覆盖。如果虚函数的返回类型是基类的指针或引用，则覆盖函数可以返回派生类的指针或引用。这些被称为协变返回类型。下面是一个例子:

#include <iostream>
 
class Base
{
public:
	// This version of getThis() returns a pointer to a Base class
	virtual Base* getThis() { std::cout << "called Base::getThis()\n"; return this; }
	void printType() { std::cout << "returned a Base\n"; }
};
 
class Derived : public Base
{
public:
	// Normally override functions have to return objects of the same type as the base function
	// However, because Derived is derived from Base, it's okay to return Derived* instead of Base*
	Derived* getThis() override { std::cout << "called Derived::getThis()\n";  return this; }
	void printType() { std::cout << "returned a Derived\n"; }
};
 
int main()
{
	Derived d{};
	Base* b{ &d };
	d.getThis()->printType(); // calls Derived::getThis(), returns a Derived*, calls Derived::printType
	b->getThis()->printType(); // calls Derived::getThis(), returns a Base*, calls Base::printType
 
	return 0;
}
/*
    called Derived::getThis()
    returned a Derived
    called Derived::getThis()
    returned a Base
*/

请注意，一些较老的编译器(例如Visual Studio 6)不支持协变返回类型。

关于协变返回类型有一个有趣的注意事项:c++不能动态地选择类型，因此您总是会得到与被调用函数的基版本匹配的类型。

在上面的例子中，我们首先调用d.getThis()。因为d是一个派生类，它调用Derived::getThis()，它返回一个派生类指针。然后使用Derived*调用非虚函数Derived::printType()。

有趣的是。然后调用b->getThis()。变量b是一个指向派生对象的基指针。Base::getThis()是虚函数，因此调用了Derived::getThis()。虽然Derived::getThis()返回一个Derived*，但是因为函数的基本版本返回base *，返回的Derived*将被向上转换为base *。因此，Base::printType()被调用。

换句话说，在上面的例子中，你只有在调用getThis()的时候才会得到一个派生类对象。
