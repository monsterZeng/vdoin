# 神经消息传递

# 概述

在GNN消息传递期间迭代期间，通过聚合每个节点v∈Vv\in\mathcal{V}v∈V的邻域N(v)\mathcal{N}(v)N(v)的信息来更新其隐特征hvkh_v^{k}h​v​k​​，过程如下

hu(k+1)=UPDATEk(hu(k),AGGERATE({hv(k),∀v∈N(u)}))\begin{aligned} h_u^{(k+1)}&=\text{UPDATE}^{k}(h_u^{(k)},\text{AGGERATE}(\{h_v^{(k)},\forall v \in \mathcal{N}(u)\}))\\ \end{aligned} ​h​u​(k+1)​​​​​​=UPDATE​k​​(h​u​(k)​​,AGGERATE({h​v​(k)​​,∀v∈N(u)}))​​

其中UPDATE\text{UPDATE}UPDATE 和AGGERATE\text{AGGERATE}AGGERATE 是任意可微的的函数(神经网络)。

k=0是节点的初始化特征。

以上是针对每个点更新的公式，下面是图级别的公式

H(k+1)=σ(AH(k)Wneigh(k+1)+H(k)Wself(k+1))H^{(k+1)}=\sigma\left(AH^{(k)}W_{\text{neigh}}^{(k+1)}+H^{(k)}W_{\text{self}}^{(k+1)}\right) H​(k+1)​​=σ(AH​(k)​​W​neigh​(k+1)​​+H​(k)​​W​self​(k+1)​​)

# 自消息闭环

通过给每个节点增加自环(已有自环的就不用在增加了)，可以不用显示地定义更新函数，如下所示

hu(k)=AGGERATE({hv(k−1),∀v∈N(u)∪{u}})h_{u}^{(k)}=\text{AGGERATE}(\{h_v^{(k-1)},\forall v \in \mathcal{N}(u)\cup\{u\}\}) h​u​(k)​​=AGGERATE({h​v​(k−1)​​,∀v∈N(u)∪{u}})

这样可以简化消息的传递，有时候还能缓解过拟合。但是，却也严重限制了GNN的表达能力，因为无法将节点领域信息与自身信息区分开而导致过平滑问题。这个问题之后会展开来讲。

# 广义邻域聚合

# 邻域归一化

最基本的邻域聚合函数仅取邻居嵌入的总和。如果邻域聚合函数对节点度高度敏感，那么会导致结果很不稳定。例如，节点uuu的邻域中节点数是节点u′u'u​′​​邻域中节点数的100倍，那么∣∣∑v∈N(u)hv∣∣≫∣∣∑v′∈N(u′)hv′∣∣||\sum_{v\in\mathcal{N}(u)}h_v||\gg||\sum_{v'\in\mathcal{N}(u')}h_v'||∣∣∑​v∈N(u)​​h​v​​∣∣≫∣∣∑​v​′​​∈N(u​′​​)​​h​v​′​​∣∣​。这种幅度的巨大差异会导致数值不稳定及优化困难。

解决方案：

基于节点度的归一化，也就是是在用均值来代替求和

mN(u)=∑v∈uhv∣N(u)∣m_{\mathcal{N}(u)}=\frac{\sum_{v\in \mathcal{u}}h_v}{|\mathcal{N(u)}|} m​N(u)​​=​∣N(u)∣​​∑​v∈u​​h​v​​​​

对称归一化

mN(u)=∑v∈uhv∣N(u)∣∣N(v)∣m_{\mathcal{N}(u)}=\frac{\sum_{v\in \mathcal{u}}h_v}{\sqrt{|\mathcal{N(u)}||\mathcal{N(v)}|}} m​N(u)​​=​√​∣N(u)∣∣N(v)∣​​​​​∑​v∈u​​h​v​​​​

是否归一化

正确的归一化能使得数值更加稳定。但是，也会导致信息丢失。可能很难使用学习到的嵌入来区分不同度的节点，并且会掩盖图的结构特征。因此，是否归一化需要根据特定的应用来决定。当节点的特征信息远比结构信息游泳或者由于节点度范围过于广泛导致优化过程可能不稳定的任务重，归一化最有用。

# 集合聚合操作

邻域聚合从根本上来说是集合函数，即给定一组邻域中的节点嵌入hv,∀v∈N(u)h_v,\forall v\in \mathcal{N}(u)h​v​​,∀v∈N(u)，必须将此集合映射到一个向量mN(u)m_{\mathcal{N(u)}}m​N(u)​​​。集合就代表着无序性，因此任何聚合函数都必须是置换不变的。

一些简单的的置换不变的聚合函数有：求和、均值、最大和最小。

# 集合池化

一种定义聚合函数的原则是基于置换不变神经网络的理论。

Deep Set结果表明，将一组嵌入映射到一个嵌入的任何置换不变函数都可以由下式模型逼近到任何精度。

mN(u)=MLPθ(∑v∈N(u)MLPϕ(hv))m_{\mathcal{N}(u)}=\text{MLP}_{\theta}\left(\sum_{v \in \mathcal{\mathcal{N}(u)}}\text{MLP}_{\phi}(h_v)\right) m​N(u)​​=MLP​θ​​​⎝​⎛​​​v∈N(u)​∑​​MLP​ϕ​​(h​v​​)​⎠​⎞​​

# Janossy池化

不使用置换不变的压缩方法，而是采用置换敏感的函数并对多种可能的置换取均值。

具体操作为：πi∈Π\pi_i \in \mathcal{\Pi}π​i​​∈Π表示将集合{hv,∀v∈N(u)}\{h_v,\forall v \in \mathcal{N}(u)\}{h​v​​,∀v∈N(u)}映射到特定序列(hv1,hv2,...,hvN(u))πi(h_{v_1},h_{v_2},...,h_{v_{\mathcal{N}(u)}})_{\pi_i}(h​v​1​​​​,h​v​2​​​​,...,h​v​N(u)​​​​)​π​i​​​​的置换函数。换句话说，πi\pi_iπ​i​​将无序的邻居嵌入集置于任意排序的序列中。然后通过Janossy池化实现领域聚合

mN(u)=MLP(1∣Π∣∑π∈Πρϕ(hv1,hv2,...))m_{\mathcal{N}(u)}=\text{MLP}\left(\frac{1}{|\mathcal{\Pi}|}\sum_{\pi\in\mathcal{\Pi}}\rho_{\phi}(h_{v_1},h_{v_2},...)\right) m​N(u)​​=MLP(​∣Π∣​​1​​​π∈Π​∑​​ρ​ϕ​​(h​v​1​​​​,h​v​2​​​​,...))

ρϕ\rho_{\phi}ρ​ϕ​​是置换敏感的函数(如应用在序列数据的神经网络，例如LSTM)。

置换函数集合Π\mathcal{\Pi}Π包含了所有可能的置换函数。实践中，对所有可能的置换求和通常很困难，所以

  • 在每次应用聚合函数时，对所有可能的置换采样出一个随机子集，并且仅对该子集进行求和
  • 对邻域中的节点进行规范化排序，例如，根据节点度对节点进行降序排序，并随机断开一些关联关系。

# 广义更新方法

# 过平滑

节点特征的过平滑问题是GNN一个核心问题。

通常来说，深层的神经网络的拟合能力是大于浅层的神经网络的。但是，在GNN中，如果聚合方法不区分出目标节点的特征与邻域节点的特征，例如：求和，求均值，最大和最小时，那么目标节点的特征会过于依赖邻域节点特征，导致深层网络下所有节点的特征都很相似。这种情况下，就不能区分节点了。

# 缓解方法

# 简单拼接

直接将基本更新函数的输出与节点的上一层表示拼接在一起。

\text{UPDATE}_{\text{concat}}(h_u,m_{\mathcal{N}(u)})=\left[\text{UPDATE}_{\text{base}}(h_u,m_{\mathcal{N}(u)})\bigoplus​h_u\right]

主要思路是鼓励模型在消息传递过程中解耦信息，将邻域聚合的信息与每个当前的表示区分开来。

# 跳跃知识连接

提高最终的节点表示质量的一种不冲策略是利用消息的每一层输出的表示，而不是仅使用最后一层。

zu=fJK(hu(0)⨁hu(1)⨁⋯⨁hu(K))z_u=f_{JK}(h_u^{(0)}\bigoplus h_u^{(1)} \bigoplus\cdots \bigoplus h_u^{(K)}) z​u​​=f​JK​​(h​u​(0)​​⨁h​u​(1)​​⨁⋯⨁h​u​(K)​​)

fJKf_{JK}f​JK​​是任意微分函数。最简单的为直接拼接。复杂的还有最大池化，或者输入到LSTM。
